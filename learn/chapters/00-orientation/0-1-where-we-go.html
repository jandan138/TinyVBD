这本书要带你去哪

我们先不讲任何公式。下面这根「头发」正在你的浏览器里被真正地仿真——它跑的就是 TinyVBD 的核心算法 Vertex Block Descent。先玩一会儿，建立一点 feeling，再开始讲为什么。

interactive先上手：一根真正在跑 VBD 的 strand
用鼠标拖动任意顶点甩它；调 stiffness（弹簧多硬）、末端质量比（末端有多重）、迭代数（每帧解多少次）。注意：即使把末端质量调到 1000×、迭代数压到很低，它也不会爆炸。这正是 VBD 的招牌。

这是什么，为什么值得读

你刚才拖动的，是一条 mass-spring strand——一串被弹簧连起来的质点，可以当作一根头发、一段绳子或一条链。TinyVBD 是一份只有 ~400 行、唯一依赖 Eigen 的 C++ 程序，它把这根 strand 一帧帧地仿真出来，存成 JSON 给 Blender 可视化。

但它不是又一个「调调弹簧、反复迭代」的玩具。它是 SIGGRAPH 2024 论文 Vertex Block Descent（VBD）的官方极简实现。那篇论文的完整版能在一块 RTX 4090 上仿真上亿自由度、百万接触的场景，且无条件稳定——哪怕每帧只迭代一次、时间步很大，也不炸。TinyVBD 把同一套思想剥到最干净的骨架：没有 GPU、没有碰撞、没有四面体，只有一根 strand。这恰恰是读懂它的最好机会。

从 PBD 看 它长得像你认识的 PBD，但内核换了

你对 PBD 的印象大概是：预测一下位置，然后把粒子硬掰到约束上、一遍遍迭代，迭代越多越「硬」。VBD 表面上也很像——它也直接更新位置、也用 Gauss-Seidel 一个个扫过去。但它不把力转成约束去硬掰，而是让每个顶点解一个小小的牛顿步，朝着「让总能量下降」的方向走。这一字之差，换来了 PBD 给不了的稳定性和物理正确性。这本书的主线，就是讲清楚这个「内核替换」。

这本书假设你已经会什么

我假设你上过 GAMES103：知道质点、弹簧、显式/隐式时间积分这些词，自己写过弹簧质点系统。我也假设你对 PBD 的理解停留在课堂例子那一层——粗暴调位置、反复迭代——还没深究过它和「真正的弹性力学」是什么关系。

我不假设你对隐式欧拉背后的数学本质很清楚。事实上，很多人写过隐式积分却说不清「它到底在求什么」。这本书会专门花一个 Part 把这件事讲透——因为它是理解 VBD（以及 PBD / XPBD / Projective Dynamics 全家）的那把钥匙。

一个语言约定

正文用中文娓娓道来，但所有架构术语、概念名、命令名、性能指标一律保留英文（Gauss-Seidel、incremental potential、Hessian、compliance…）。这是为了让你读完能无缝衔接论文和代码，而不是记一套只在中文语境里成立的译名。

路线图：我们会怎么走

这本书有一条精心安排的坡度，建议顺着读：

  • Part 1 · 数学地基：补回那块发虚的拼图——隐式欧拉的「variational 真身」：一步隐式欧拉 = 最小化一个能量 $G(\mathbf x)$。这是全书的地基。
  • Part 2 · 谱系：把 PBD / XPBD / Projective Dynamics 摆在一起，发现它们其实都在解同一个 $\min G(\mathbf x)$，只是「怎么解」不同。你对 PBD 的旧印象会在这里被校准。
  • Part 3 · VBD 核心思想：block coordinate descent——一次只动一个顶点。为什么「局部下降」能保证「全局下降」，为什么它又快又稳。
  • Part 4 · 读代码：把 TinyVBD 的 forwardStep() / solve() / Chebyshev 加速逐行读透，每一行都对得上前面的公式。
  • Part 5 · 实验：亲手复现论文里的招牌实验——高质量比、高刚度比、弯曲——并理解每个实验在「考」VBD 的什么。
  • Part 6 + 附录：VBD 的位置、局限与后续，外加术语表、公式速查、参考文献。

每一章都尽量配一个可交互的小实验。它们不是装饰——很多直觉用文字讲十句，不如你拖一下滑块看一眼。

这一节你要带走的

就三件事：(1) TinyVBD 是 VBD 论文的极简 strand 实现，麻雀虽小五脏俱全；(2) VBD 表面像 PBD，但内核是「让每个顶点解一个牛顿步、把总能量推下去」，而不是硬掰约束；(3) 理解它的钥匙，是先理解「隐式欧拉 = 最小化一个能量」。
