高质量比：XPBD 为何崩、VBD 为何不崩

这是 TinyVBD 的默认实验，也是 VBD 论文最响亮的招牌之一。一根又轻又细的链子，末端挂着一个比每节链都重 1000 倍的重锤。对 XPBD 这是一道送命题；对 VBD 它几乎不费力。这一节我们把「为什么」讲透。

实验设置：1000× 重锤吊在轻链上

跑起来就是 initializeTilted()，initialize() 里默认调的就是它。参数全部来自真实代码：

  • 20 个顶点，间距 dis = 0.05，初始 initHeight = 2.0，整条链按 tanAngle = 0.57735 即 30° 向上倾斜摆放；
  • stiffness = 1e8——极硬的弹簧，硬到近乎不可拉伸；
  • 质量：前 19 个顶点 m0 = 1，最末端那个顶点 m1 = 1000——质量比正好 1000×；
  • skip spring 打开（addSkipSpring = true，skipSpringStrength = 100），给链子一点抗弯刚度，见 5.3；
  • 第 0 个顶点被 pin 住（mVelocity.col(0).setZero()），整条链悬吊在它下面。

把它想象成：一根近乎不可拉伸的细绳，绳头钉在天花板，绳尾系着一个千倍重的铁球。重力一拉，所有张力都集中到那些又轻又硬的链节上。这正是一道压力测试：质量分布极不均匀，弹簧又硬到求解器最难伺候。

为什么高质量比是道难题

难点不在「硬」，而在「质量比悬殊」。一个 1000× 的重锤吊在 1× 的轻链下面，意味着求解器在同一步里要同时照顾两种截然不同的尺度：重锤几乎不动，轻链却要被它狠狠地拽。谁来分配「张力」，决定了求解器会不会崩。这里 primal 与 dual 两种解法分道扬镳。

从 PBD 看 XPBD 按逆质量分配修正，重物纹丝不动

回忆 2.2：XPBD 是 dual（对偶）求解器，它解的是约束力 / Lagrange multiplier。每处理一个约束，它把位置修正按逆质量 $w_i = 1/m_i$ 分配给两端：$\Delta\mathbf x_i = w_i\nabla_i C\,\Delta\lambda$。重锤的 $w = 1/1000$ 小得可怜，轻链的 $w = 1$ 大一千倍——于是同一个 $\Delta\lambda$ 几乎全砸在轻链那一端，重锤纹丝不动。

把 1:1000 的位移分配算给你看

上面那句「同一个 $\Delta\lambda$ 几乎全砸在轻链那一端」是能一行算出来的。取那根连着重锤的弹簧，一端是轻链顶点（$m_\text{light}=1$，逆质量 $w_\text{light}=1$），另一端是重锤（$m_\text{heavy}=1000$，逆质量 $w_\text{heavy}=1/1000$）。XPBD 一次约束投影把两端各挪：

$$\Delta\mathbf x_\text{light} = w_\text{light}\,\nabla_\text{light}C\,\Delta\lambda,\qquad \Delta\mathbf x_\text{heavy} = w_\text{heavy}\,\nabla_\text{heavy}C\,\Delta\lambda$$

两端的约束梯度 $\nabla C$ 是同一条弹簧的两个方向，大小相等（$\lVert\nabla_\text{light}C\rVert=\lVert\nabla_\text{heavy}C\rVert$），$\Delta\lambda$ 又是两端共用的同一个标量。所以两端位移大小之比只由逆质量比决定：

$$\frac{\lVert\Delta\mathbf x_\text{light}\rVert}{\lVert\Delta\mathbf x_\text{heavy}\rVert} = \frac{w_\text{light}}{w_\text{heavy}} = \frac{1}{1/1000} = 1000$$

读出来：这一次投影把弹簧的长度误差「修」掉，轻端动了 $1000$ 份、重端只动了 $1$ 份——重锤几乎纹丝不动，修正几乎全压在轻链上。这正是「dual 求解器按逆质量打折」的算术：质量比 $1000\times$，位移分配就是 $1000:1$。要把张力真正传到重端、让重锤跟着走，就得靠极多次 sweep 一点点累积——这就是高质量比下 XPBD 收敛极慢、有限迭代里轻链被拉爆的根。

这组数是为讲解构造的自洽示例

$1000:1$ 这个比值用本实验真实的 $m_0=1$、$m_1=1000$ 直接代出来、可复算；它只刻画单次约束投影内两端的瞬时分配，不是整步的最终位移（最终位移要把多条约束、多次 sweep 的累积都算进去）。要记的趋势是：逆质量比 = 位移分配比，质量越悬殊重端越「锁死」。这是 dual 求解器的结构性质，和刚度设多大无关（见下一个 callout）。

这就是病根。把张力写出来看：链节越靠近绳头，它下面挂的总重量越大，需要承受的张力也越大。但 XPBD 的逐约束 Gauss-Seidel 在一次 sweep 里只能把每个约束局部地推一点点，而每一点修正又被逆质量按比例「打折」到轻的那一端。结果是重锤这一侧的修正被严重低估，而轻链被反复抽打、过度拉伸。更要命的是：哪怕你把 compliance 设成 0（无穷硬，退回 PBD），也救不回来——

无穷大的刚度也压不过质量比

直觉上你会觉得「弹簧够硬就不会被拉长」。但在 dual 求解器里，刚度只决定约束想把误差推到多小，质量比却决定每次迭代能推动多少。质量比悬殊时，迭代收敛极慢，有限迭代根本到不了那个「硬」的解——于是轻的一侧被拉爆、整条链像被甩出去一样。刚度无穷大，也无法弥补质量比带来的病态。这正是 dual 求解器在高质量比下崩掉的本质。

VBD 为什么稳如老狗：primal 直接平衡力

VBD 是 primal 求解器——它不解约束力，而是直接对每个顶点求位置。回忆 1.4 和 Part 3：在每个顶点上，VBD 做一步局部牛顿，平衡的是完整的局部力，其中包含那一项惯性力：

$$\mathbf f_i = -\frac{m_i}{h^2}(\mathbf x_i-\mathbf y_i)\;-\;\sum_{j\in\mathcal F_i}\frac{\partial E_j}{\partial\mathbf x_i},\qquad \mathbf H_i = \frac{m_i}{h^2}\mathbf I + \sum_{j\in\mathcal F_i}\frac{\partial^2 E_j}{\partial\mathbf x_i^2}$$

关键在惯性项里的系数 $m_i/h^2$。对那个 1000× 的重锤，这一项同样被放大 1000 倍：它的 Hessian 对角线很大、它对偏离预测位置 $\mathbf y_i$ 的「抵抗」也很大。换句话说，重锤的质量不是被当成「逆质量权重去打折修正」，而是实打实地进了它自己那个 $3\times3$ 牛顿系统。重的顶点解出来该往哪走，就稳稳地往哪走；轻链顶点解它自己的小系统，把张力直接、对称地平衡掉。质量比在这里不是病态来源，只是各个顶点 Hessian 上不同大小的数字而已。

从 GAMES103 看 这其实就是隐式欧拉本该有的样子

你在 GAMES103 里写的隐式欧拉，组装的全局矩阵就是 $M/h^2 + \nabla^2 E$，质量本来就在对角线上、本来就该这么平衡力。VBD 只是把这件事拆成逐顶点的 $3\times3$ 小问题来做（block coordinate descent），但没有像 XPBD 那样把质量降格成逆质量权重。所以它继承了隐式欧拉对质量比的天然鲁棒——高质量比对它根本不构成压力。

JS 端复现：1000× 重锤只拉伸约 1%

本教程里那个能拖动的 strand-lab（见 0.1 与 5.4）是 VBD 核心的 JavaScript 移植，它复现了同样的现象：把末端质量比推到 1000×、弹簧设得很硬，整条链在重力下也只过度拉伸约 1%，保持紧绷，不会被甩飞。这正是 primal 求解器面对高质量比的从容。

这一节你要带走的

三句话：(1) 默认实验 initializeTilted() 是一道高质量比压力测试——20 节轻硬链吊一个 1000× 重锤；(2) XPBD 是 dual 求解器，按逆质量分配修正，重物压垮轻物，连无穷大刚度也救不回；(3) VBD 是 primal 求解器，质量直接进每个顶点的牛顿系统（$m_i/h^2$），高质量比对它毫无压力，1000× 重锤也只拉伸约 1%。下一节 5.2 我们诚实地翻到硬币另一面：VBD 也有它真正怕的东西——高刚度比。VBD 在整个谱系中的最终定位见 6.1。
