你能跑的实验菜单

前面三节把每个旋钮背后的数学讲清楚了。这一节是一张动手清单：TinyVBD 里所有能跑的实验，都藏在 main.cpp 的几个函数和一堆被注释掉的 config 里。改一两行、重新编译，就能复现论文里的招牌场景。最后，去 capstone 模拟器里亲手把所有旋钮跑一遍。

总开关：initialize() 里换一行

整个实验的入口就在 initialize()。它默认调 initializeTilted()；想换实验，把它注释掉、取消另一个的注释即可：

main.cpp · initialize()
void initialize() {
    initializeTilted();        // 默认：高质量比（5-1）
    //initializeStiffRatio();  // 取消注释 → 高刚度比（5-2）
    // 也可改成 initializeHorizontal(); 自己加一行

    outPath = params.outPath;
    createFolder(outPath);
}

三个 initialize 函数各自构造一种 strand。它们之间的对比，本身就是一组对照实验：

函数它设置了什么它在演示什么
initializeTilted()
（默认）		 20 顶点，dis=0.05，stiffness=1e8，质量 1 / 末端 1000，30° 倾斜，skip spring 开（强度 100） 高质量比：1000× 重锤吊轻链，VBD 保持紧绷（约 1% 拉伸），XPBD 会崩 → 5.1
initializeStiffRatio() 5 顶点，刚度按 iV%2 交替 1e4 / 1e8（1:10000），质量均匀 0.1，skip spring 开 高刚度比：VBD 的软肋，软段过拉、硬段锁死，收敛慢 → 5.2
initializeHorizontal() 3 顶点，dis=0.1，stiffness=1e6，质量 1 / 末端 1000，水平摆放，无 skip spring 最小的高质量比场景：3 个点就能看出重锤把张力拉给哪一节，适合配 vbd-sweep 慢放看逐顶点更新

进阶：取消注释那些 config 变体

initializeStiffRatio() 顶部有一整叠被注释掉的配置块——它们不是死代码，而是论文不同图表对应的设置。取消注释（并注释掉当前生效那块）就能切换：

变体（顶点 / 迭代 / ρ）它设置了什么它在演示什么
3 verts / 100 / ρ=0.0最短的刚度比链，无加速baseline：100 迭代下软硬链还没收敛到位
3 verts / 20000 / ρ=0.0
_converged		把迭代拉到 20000真收敛需要多少迭代——和上一行对比，看软段从过拉到绷紧
10 verts / 100 / ρ=0.08
_accelerated		更长的链 + Chebyshev 加速Chebyshev（4.4）能缓解慢收敛，但治标不治本
10 verts / 20000
_accelerated_converged		长链跑到真收敛长链上高刚度比更吃力，信息要传更远
5 verts / 20000 / ρ=0.0
_converged		当前 5 顶点链的收敛版和默认 5 verts / 100 对比，量出收敛差距
跑出来该长什么样：100 迭代 vs 20000 迭代

表里 _converged 行只说「真收敛要多少迭代」，但没告诉你屏幕上该看到什么差别、怎么判断「还没收敛」。给一句可观测判据：在那条高刚度比链上跑 100 迭代，你会看到 软段（1e4）被重力过度拉长、硬段（1e8）几乎纹丝不动——残余的约束违反肉眼可见，软段明显比 rest length 长一截。把迭代拉到 20000，软段才被一点点绷回接近 rest length，整条链稳定下来。

关键是：这个「过拉 + 迟钝」不是不稳定。能量 $G$ 在两种迭代下都始终在下降，链既不爆也不抖——差的只是精度没到，迭代被高刚度比拖住、信息在软硬交界处传得太慢（5.2）。这正对应 6.1 要点破的那句「无条件稳定 ≠ 无条件正确」：稳定保证它不炸，但不保证 100 迭代就解对。看到软段过拉别急着以为求解器坏了——多半只是迭代不够。

另外还有几个旋钮散落在各处，都可以单独拨动：

  • params.numIterations（默认 100）：每帧解多少次。压低看数值阻尼变大、链变软，但永不爆（5.3）。
  • params.substeps（默认 1）：把一帧切成几段，主循环里 dt/=substeps。源码 outPath 注释里那个 10substeps 变体就是把它设到 10——更小的 $h$ 让 $G$ 更好解。算一笔：惯性项权重是 $1/h^2$，$h\to h/10$ 让 $1/h^2$ 涨 $100\times$（$3600\to3.6\times10^5$），惯性碗陡 100 倍 → 局部牛顿步条件数改善、$G$ 更凸更好解。「更小 $h$ 更好解」落到的就是 $1/h^2$ 的这个具体倍数。
  • params.useAcceleration + params.accelerationRho：开关 Chebyshev 加速并调谱半径估计 ρ。ρ 太大会 overshoot/振荡（4.4）。
  • addSkipSpring / skipSpringStrength：开关 / 调弯曲刚度。关掉看链如何自折叠卷曲（5.3）。
  • 刚度比 stiffness1 / stiffness2（stiffnessRatio48 那一族）：把 1e4 / 1e8 改近一点，看收敛变快；改远一点，看 VBD 更吃力。
一个真实的小坑：输出路径是 Windows 风格

各 initialize 函数里的 params.outPath 都是 C:\Data\... 这样的 Windows 路径，而 createFolder 只在 _WIN32 下真正建目录。在 Linux / macOS 上跑，记得把 outPath 改成你系统的有效路径，否则 JSON 帧写不出来。这点和算法无关，纯粹是 demo 的 IO 细节。
