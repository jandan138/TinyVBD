skip spring = 弯曲，与迭代数的影响

这一节调两个旋钮，都不改求解器、只改「问题」本身。一个是 skip spring——一根连 i 与 i+2 的弹簧，给本来软趴趴的链一点抗弯刚度；另一个是 numIterations——每帧解多少次，决定你离隐式欧拉真解有多近。

纯距离弹簧的链没有弯曲刚度

一条只有相邻距离弹簧（i 连 i+1）的链，有一个你可能没注意到的问题：它完全不抵抗弯折。每根距离弹簧只在乎两端的长度，对「链拐了个弯」没有任何意见。于是这种链在重力下会瘫成一团、卷曲、自己折叠——因为把它对折，所有距离弹簧的长度都没变，能量一点没涨，求解器没有任何力去把它撑开。一根真实的头发或绳子不会这样，它有弯曲刚度（bending stiffness）。

从 GAMES103 看 你大概加过 bending 约束

课上做布料 / 绳子时，你多半在相邻距离弹簧之外，额外加过一层「弯曲约束」——比如连接隔一个的顶点，或约束二面角。没有它，布会软成一摊。skip spring 就是这套思路里最便宜的一种弯曲能量：不算二面角，只多加一根跨一个顶点的距离弹簧。

skip spring：连 i 与 i+2 的一根便宜弹簧

看 Strand.h 里带 skip spring 的那个 from(...)。在为每条相邻边（i, i+1）建好弹簧后，只要 addSkipSpring 为真，它就顺手再建一根 (i, i+2) 的弹簧：

Strand.h · from()
vertAdjacentEdges[i].push_back(edgeId);
vertAdjacentEdges[i+1].push_back(edgeId);
edgesStiffness.push_back(stiffness);
lengthesNew.push_back(lengthes[i]);
if (addSkipSpring && i < lengthes.size() - 1)
{
    edgeId = edges.size();
    edge << i, i + 2;                          // 跨过 i+1，连 i 和 i+2
    edges.push_back(edge);
    vertAdjacentEdges[i].push_back(edgeId);
    vertAdjacentEdges[i + 2].push_back(edgeId);
    edgesStiffness.push_back(stiffnessSkipSpring);  // 默认 100，比主弹簧软得多
    lengthesNew.push_back(lengthes[i] + lengthes[i+1]); // rest = 两段之和
}

三个细节值得品：(1) 这根弹簧用单独的 stiffnessSkipSpring（默认 100），通常远比主距离弹簧软；(2) 它的 rest length 取 lengthes[i] + lengthes[i+1]，也就是两段直接相加——意味着「链是直的」时它恰好等于 rest、不出力；(3) 一旦链在 i+1 处弯折，i 到 i+2 的直线距离就变短，这根弹簧被压缩、产生一个把链撑直的恢复力。这就是它近似弯曲能量的全部机制：用「跨节点距离」间接惩罚弯折。对 VBD 的求解器而言，它不过是 vertAdjacentEdges 里多出来的一条边，solve() 照常累加它的力和 Hessian（见 4.3），不需要任何特殊处理。

这就是 initializeTilted() 里 skipSpringStrength = 100 的来历

默认实验 5.1 打开了 skip spring，正是为了让那根吊重锤的链看起来像根有筋骨的绳，而不是一摊会自折叠的距离弹簧。initializeStiffRatio() 同样开了它。

迭代数：离真解多近，与「数值阻尼」

第二个旋钮是 numIterations（默认 100）。回忆主循环（4.5）：每个 substep 里，VBD 把 solve() 这个 Gauss-Seidel sweep 重复 numIterations 次。每多迭代一次，当前位形就更接近增量势能 $G$ 的真正最小值，也就是更接近隐式欧拉的真解。

  • 迭代多：更贴近真解，链更「绷」、更硬挺，人为的能量损耗少；
  • 迭代少：还没走到谷底就停了，表现为系统偏软、像是被加了一层数值阻尼（numerical damping）——运动被悄悄吃掉一点，看起来更黏、更慢。

但无论迭代多少，有一条铁律：VBD 不会爆炸。因为每个局部牛顿步都让 $G$ 单调下降（Part 3），哪怕只迭代 1 次，能量也只降不升——结果是「软一点、阻尼大一点」，绝不会是「发散」。这正是 VBD 招牌的无条件稳定：你可以为了省算力把迭代压到很低，最坏情况只是损失精度和一点弹性，换不来一次爆炸。

从 PBD 看 同样是「迭代越多越硬」，含义却不同

PBD 里「迭代越多越硬」是个病：有效刚度随迭代数漂移，stiffness 成了 solver 设置的产物而非物理量（2.1）。VBD 里「迭代越多越接近真解」是个特性：它收敛到的是一个良定义的目标——隐式欧拉解。刚度是你设的物理量 $k$，迭代只决定你离那个固定的解有多近，不会改变解本身。

迭代 vs substeps：另一种花预算的方式

除了在一个时间步内多迭代，你还能把时间步切细：主循环开头 params.dt /= params.substeps，把一帧拆成多个 substep，每个 substep 用更小的 $h$ 跑一遍完整的 forwardStep + 迭代 + updateVelocity。更小的 $h$ 让惯性项 $m/h^2$ 的系数更大、$G$ 更「凸」更好解，也让每个子步的运动更短、更线性。所以你手里其实有两种花算力的方式：同一个 $h$ 多迭代，或把 $h$ 切细成多 substep。两者都让结果更接近真解，权衡在于哪种对你的场景更划算——这也是 5.4 实验菜单里值得亲手试的旋钮（比如源码注释里那个 10substeps 变体）。

本节不放新部件——想实时把玩 skip spring 与迭代数，直接去 0.1 或 5.4 的 strand-lab：拖动链、调迭代滑块，亲眼看迭代变少时链如何变软、关掉弯曲时链如何卷起来。

这一节你要带走的

三句话：(1) skip spring 连 i 与 i+2、rest length 取两段之和、用很软的 stiffnessSkipSpring（默认 100），是一种最便宜的弯曲能量——没有它，纯距离弹簧的链会自折叠卷曲；(2) numIterations 越多越接近隐式欧拉真解、越硬挺，越少越软、数值阻尼越大，但 VBD 哪怕 1 次迭代也稳不发散；(3) 切 substep（dt/=substeps）是另一种逼近真解的方式。下一节 5.4 给你一张完整的实验菜单，把所有旋钮和 main.cpp 的 config 对上号。
