MPM Lite 主讲：线性核、无粒子求积

这是 MPM 篇的主讲章。前面我们把 MPM 一步循环（13.2）、传输（13.3）、粒子上的 $F_p$ 与本构（13.4）、以及隐式 MPM = 网格上的 $\min G(x)$（13.5）都铺好了，也指明了一个痛点：隐式 MPM 的求解开销大致正比于每格粒子数（PPC）× 宽核 stencil。MPM Lite 正面回答这个痛点，它的核心主张可以浓缩成一句：让粒子主要只作运动状态（kinematic state）的载体，求解时不再做基于粒子的求积（quadrature），改用线性核。这一节我们照 12.7/12.8 的黄金做法，给它配一段手算 + 一段伪代码走读，把「为什么这样能省」看到机制层。

关于前沿的诚实声明：只述机制、不臆造数字

MPM Lite 是较新的工作（据其项目页 mpmlite.github.io，副题大意为「线性核、无粒子积分」）。本节只描述其机制——线性核、去掉基于粒子的求积、把粒子定位为运动状态载体——这些是可以讲清的算法思想。但本节不给出具体加速倍数、网格规模、精确能量泛函或作者列表等需要逐字核对的数值/事实：凡不确定处一律标「据项目页…／机制上…」。下面的手算同样是为讲解构造的自洽示例，非论文复现：能写准的（核宽、每粒子节点数、计数比例）才较真，会随实现/参数缩放的（绝对开销、加速比）一律标为示意。重点始终是「省在哪、怎么省」的机制，不是某个具体小数。

先定位开销：PPC × 宽核 stencil

要懂 MPM Lite 省在哪，先把标准隐式 MPM 的开销摊开。隐式 MPM（13.5）每步要在网格节点上解一个 $\min G(x)$，弹性能那一项写成对所有粒子求和：

$$G_{\text{elastic}}(x_g)=\sum_{p}V_p^0\,\psi\bigl(F_p(x_g)\bigr).$$

这条求和有两层放大因子，正是开销的来源：

  • PPC（particles per cell，每格粒子数）：标准做法把每个粒子都当成一个求积点（quadrature point）来累加能量、力、Hessian。一格里塞 8 个、甚至更多粒子是常态，于是同一块区域的能量被 PPC 份地重复采样、累加。
  • 宽核 stencil：经典 MPM 为了光滑稳定用二次/三次 B-spline 核（13.2），一个粒子在 3D 里触及 $3^3=27$ 个网格节点。每个粒子的力/Hessian 要散到这 27 个节点上，Hessian 块之间的耦合也按这个宽度铺开——网格节点 $\mathbf H$ 的非零结构因此很密，稀疏求解很贵。

两者一乘：每步隐式求解的装配与求解成本 $\sim$ 粒子数 × 每粒子节点数，而粒子数 = 格数 × PPC、每粒子节点数 = 核宽$^{\text{维数}}$。MPM Lite 的两刀正好砍这两个因子：去粒子求积砍掉 PPC 那层重复，线性核把每粒子节点数从 $3^d$ 降到 $2^d$。

interactive切核看「每粒子触及多少网格节点」
切到 linear（MPM Lite 用），看一个粒子只覆盖每维 2 个节点（3D 共 $2^3=8$）；切到 quadratic B-spline（经典 MPM），每维 3 个、3D 共 $3^3=27$。节点数正比于隐式求解的 stencil 开销——这张图就是 MPM Lite「线性核」那一刀的直观依据。（compact(CK) 是 13.8 的另一条路：用 $C^2$ 紧核在节点数接近线性的同时保住光滑。）

MPM Lite 的两刀：去粒子求积 + 线性核

第一刀 · 粒子降级为运动状态载体。标准 MPM 里粒子身兼两职：既是运动状态的载体（位置、速度、$F_p$——这是 Lagrangian 的本分），又是求积点（求解时在它身上采样能量/力/Hessian）。MPM Lite 据项目页的机制是把这两职拆开：粒子继续忠实地携带运动状态（这部分 Lagrangian 的优势——巨形变、多材料、历史 $F_p$——一样不丢），但在求解能量与力时不再以粒子为求积点。少了「按 PPC 份在每个粒子上重复采样」这层，装配的求和规模随之大幅收缩——这是去掉 PPC 那个放大因子的来源。

第二刀 · 线性核。求解时改用线性核（每维 2 节点、$C^0$），而非经典的二次/三次 B-spline。每粒子（或每求积单位）触及的网格节点从 $3^d$ 降到 $2^d$——3D 里 27 → 8，stencil 更窄，网格 $\mathbf H$ 的非零更少、耦合更局部，稀疏求解显著变便宜。线性核本身精度/稳定性弱（只有 $C^0$），但 MPM Lite 据机制是靠「重新组织求积、去掉粒子重复采样」等手段把这部分补回来——这也是它和单纯「退回线性核」截然不同的地方。

保住了什么：MPM 的多材料通用性

关键是这两刀没动本构。粒子照旧携带 $F_p$，能量照旧是 9.3 谱里的 $\psi(F_p)$、应力照旧 $P_p=\partial\psi/\partial F_p$（13.4）——所以 MPM「弹性体/雪/沙/泥/可压流体一套框架通吃」的多材料卖点原样保留。MPM Lite 改的是「怎么把能量求积、把力散到网格、解多宽的 stencil」这层离散与求解机制，不是「材料是什么」。据项目页，它在隐式上带来明显加速，显式情形也有改善——具体倍数我们不写（DO-NOT-STATE），但省的来源（PPC 与核宽两个因子被砍）是机制上能讲清的。

手把手算一遍：两刀各省下多少「计数」

照 12.7 的做法，摆一组能写在一行里的数，把「省在哪」算成具体计数。声明在前：这是为讲解构造的自洽示例，非论文复现；我们只对「核宽、节点数、计数比例」这类能写准的较真，不碰绝对耗时/加速比。

设一个 3D 场景，取一小块 $4\times4\times4=64$ 格的区域，PPC = 8（每格 8 个粒子）。则这块区域的粒子数：

$$N_p = 64\times 8 = 512\ \text{个粒子}.$$

标准隐式 MPM（二次 B-spline + 粒子求积）。每个粒子触及 $3^3=27$ 个网格节点，每个粒子都是一个求积点，于是「粒子→节点」的贡献条目数（衡量装配/耦合规模的代理量）：

$$C_{\text{std}} = N_p\times 27 = 512\times 27 = 13\,824\ \text{条粒子-节点贡献}.$$

MPM Lite 的第二刀（换线性核）。每粒子触及节点从 27 降到 $2^3=8$：

$$C_{\text{lin}} = N_p\times 8 = 512\times 8 = 4\,096\ \text{条},\qquad \frac{C_{\text{lin}}}{C_{\text{std}}}=\frac{8}{27}\approx 0.30.$$

光是把核从二次换成线性，每粒子的贡献条目就降到约 三成（$8/27$）。这一步是纯几何、能写准——它就是上面 widget 里 27 → 8 那个对照的算术化。

MPM Lite 的第一刀（去粒子求积）。这一刀动的是求积点的个数那个因子。标准做法把全部 512 个粒子都当求积点；MPM Lite 据机制不再以粒子为求积点，求积单位的数量改由网格分辨率（而非 PPC × 格数）主导。用我们这块区域示意：若求积规模回到「按格」的量级而非「按粒子」的量级，那个 ×8（PPC）的放大就被摘掉。把两刀叠起来，作为示意的合并效果：

$$\underbrace{8}_{\text{PPC 因子（第一刀摘掉）}}\ \times\ \underbrace{\frac{27}{8}}_{\text{核宽因子（第二刀从 27 降到 8）}}\ \Longrightarrow\ \text{两个放大因子被同时压低}.$$
这些数是示意，趋势才是真的

$64$ 格、$\text{PPC}=8$、$27$、$8$ 这些数是为讲解挑的、能写准的（核宽与维数决定的几何计数）；而「两刀叠起来净省多少」是示意——真实加速取决于实现、求解器、稀疏结构、材料，本节不编具体倍数（DO-NOT-STATE）。能确定的趋势是：(1) 线性核把每粒子节点数按 $8/27\approx0.30$ 收缩（几何事实）；(2) 去粒子求积摘掉 PPC 那个放大因子（机制）；(3) 两者作用在同一条求和的两个独立因子上，所以效果相乘、共同把隐式求解的 stencil 与装配规模压下来。我们较真的是「哪两个因子被砍、为什么相乘」，不是某个加速数字。

伪代码走读：MPM Lite 一步循环的结构

照 12.8 的做法，把 MPM Lite 一步的结构用伪代码摆出来，再逐段对回机制。先看读法声明：

读法说明：伪代码，非真实 MPM Lite 源码，仅示意结构

下面这段是为讲清结构而写的 Python 风格伪代码，不是 MPM Lite 的真实实现，变量名/分段只为对应机制、不代表项目里的命名或精确公式。真实实现里每一行都对应大量优化过的 GPU/并行代码。我们要的是「手指能点到每一步、看清两刀落在哪」的骨架。关键差异处会标出它和标准隐式 MPM（13.5）的不同。

pseudocode · MPM Lite 一步（示意结构）
# 伪代码，非真实 MPM Lite 源码，仅示意结构
# 粒子 p 携带：位置 x_p、速度 v_p、形变梯度 F_p（永久 Lagrangian 状态）
# 背景网格每步清零重建（scratch）

def mpm_lite_step(particles, grid, dt):
    grid.clear()                                   # 网格是 scratch，每步清零

    # 1) P2G：用【线性核】把粒子质量/动量散到网格
    #    线性核 → 每粒子只触及 2^d 个节点（3D=8），不是二次核的 3^d=27
    for p in particles:
        for i in linear_neighbors(p):              # ← 第二刀：线性核，窄 stencil
            w = linear_weight(i, p)                # C0 线性权重（便宜）
            grid.m[i] += w * p.mass
            grid.mv[i] += w * (p.mass * p.v)       # APIC 可加仿射项；此处略

    # 2) 隐式 grid solve：在网格节点 DOF 上 min G(x)（同 13-5 的变分骨架）
    #    关键：能量/力的求积【不再逐粒子】，而按网格(线性核)组织 → 摘掉 PPC 放大
    def assemble_GHf(xg):                           # ← 第一刀：去粒子求积
        f, H = inertia_terms(grid)                 # 惯性项：网格节点质量，恒正定
        for q in grid_quadrature(grid):            # 按网格求积，规模 ~ 格数（非 粒子数）
            Fq = deformation_at(q, xg, particles)  # F 仍源自粒子携带的 F_p（13-4）
            P  = dpsi_dF(Fq)                        # 本构复用 FEM 篇 9-3/9-4，多材料不变
            Hq = spd_project(d2psi_dF2(Fq))        # SPD 投影（10-2）保下降方向
            scatter_to_grid(f, H, P, Hq, linear_grad)  # 用线性核梯度散到网格节点
        return f, H

    xg = grid.x0
    while not converged:                           # 牛顿迭代（同 12-8 的装配→解→线搜）
        f, H = assemble_GHf(xg)
        p_dir = sparse_solve(H, f)                 # H 因线性核更稀疏 → 解更便宜
        alpha = line_search(xg, p_dir)             # Armijo 式能量下降线搜索
        xg += alpha * p_dir
    grid.v = (xg - grid.x0) / dt                   # 由位移增量回出网格速度

    # 3) G2P：粒子从网格收新速度 + ∇v_p（同样用线性核），更新 F_p
    for p in particles:
        v_new, grad_v = gather_linear(p, grid)     # 线性核插回
        p.v = v_new
        p.F = (I + dt * grad_v) @ p.F              # F_p 增量更新（13-4），粒子随身累积

    # 4) advect：粒子按新速度移动
    for p in particles:
        p.x += dt * p.v

逐段对回机制，重点盯两刀落在哪：

  • ① P2G（线性核）——linear_neighbors/linear_weight 就是第二刀：每粒子只触及 $2^d$ 个节点，散布便宜、stencil 窄。对照 13.2 经典 MPM 这里用的是二次 B-spline 的 $3^d$ 邻域。
  • ② 隐式 grid solve（去粒子求积）——grid_quadrature 那行是第一刀的题眼：求积不再逐粒子（不是 for p in particles），而按网格组织、规模随格数而非粒子数，于是 PPC 这个放大因子被摘掉。但循环骨架——assemble → sparse_solve → line_search——和 12.8 的 IPC 外迭代、和隐式 MPM（13.5）一模一样，连 SPD 投影（10.2）都在原位。变的是求积怎么组织、核多宽，不变的是 $\min G(x)$ 的牛顿求解骨架。
  • ③ G2P + $F_p$ 更新——粒子收速度、$F_p^{n+1}=(I+\Delta t\,\nabla v_p)F_p$ 完全是 13.4 那套，只是插值核换成线性。粒子作为运动状态载体的本分一点没少。
  • ④ advect——粒子按新速度移动，和 13.2 标准循环的第四步相同。
从 TinyVBD 看 还是那个 $\min G(x)$，只是把求积和核做轻了

把这段伪代码和 12.8 的 IPC 外迭代、4.3 的 VBD solve() 并排：目标完全相同，都在最小化同一个增量势能 $G$；牛顿装配、解方向、线搜索、SPD 投影都在。MPM Lite 没有改写「优化什么」，它改写的是「怎么把这个优化的能量项更省地求积、用多宽的核散到网格」——把求积从「逐粒子 × 宽核」减负成「按网格 × 线性核」。主线一句话再次兑现：一切都是 $\min G(x)$，MPM Lite 只是让解这个 $\min G(x)$ 的每一步更轻。

这一节你要带走的

三句话：(1) MPM Lite 的两刀砍的是隐式求解开销的两个放大因子——去基于粒子的求积摘掉「每格粒子数 PPC」那层重复采样（粒子降级为纯运动状态载体），线性核把每粒子触及的网格节点从 $3^d$ 降到 $2^d$（3D 27→8、约 $0.30$）；两因子作用在同一条能量求和的不同位置，效果相乘。(2) 它没动本构——粒子照旧携带 $F_p$、能量照旧是 9.3 的 $\psi(F_p)$，所以 MPM 的多材料通用性原样保留，改的只是求积与核这层离散/求解机制；据项目页隐式显著加速、显式也改善（具体倍数本节不写）。(3) 伪代码骨架和隐式 MPM（13.5）、IPC 外迭代（12.8）逐段同构——还是那个 $\min G(x)$ 的牛顿求解，只是把求积和核做轻了。

MPM Lite 走的是「把求积和核做轻」这条路。下一条现代 MPM 路线完全换了个哲学：PB-MPM 把 MPM 重写成 position-based 的约束求解，像 PBD 那样掰位置满足约束、换大稳定步长——把我们在 2.1/2.2 学的哲学搬到 material point 上。去 13.7（PB-MPM：把 MPM 写成约束求解）。
