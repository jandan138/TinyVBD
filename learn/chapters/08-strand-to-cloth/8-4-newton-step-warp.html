per-vertex Newton step in Warp

你已经在 4.3 一行一行看过 TinyVBD 的 solve()：扫每个顶点，给它打底惯性项，再把邻接弹簧累加进 $\mathbf f_i$、$\mathbf H_i$，最后解一个 $3\times3$。现在我们看 Newton 在 GPU 上怎么做同一件事。打开 particle_vbd_kernels.py 里的 solve_elasticity kernel，你会有种奇异的熟悉感——惯性那两行几乎是从 solve() 抄过来的，只不过这回是 3D、跑在一个 Warp kernel 里，每个 thread 负责一个顶点。

同一个形状，只是搬上了 GPU

全篇主线再说一次：$\min G(\mathbf x)$ 没变。变的只是「一根 strand 串行扫 → 一块 cloth 在 GPU 上按 color 并行扫」。落到单个顶点的那一步牛顿，公式和 3.3 完全一致——力是局部能量的负梯度，Hessian 是它的二阶项，两者都拆成「惯性块 + 邻接元件块」：

$$\mathbf f_i = \underbrace{\frac{m_i}{h^2}(\mathbf y_i-\mathbf x_i)}_{\text{惯性}} + \sum_{j\in\mathcal F_i}\mathbf f_i^{(j)},\qquad \mathbf H_i = \underbrace{\frac{m_i}{h^2}\mathbf I_3}_{\text{恒正定}} + \sum_{j\in\mathcal F_i}\mathbf H_i^{(j)}$$

区别只在 $\mathcal F_i$ 这个「邻接 force elements 集合」里装的是什么。TinyVBD 里它只有弹簧（vertAdjacentEdges）；Newton 里它可能同时挂着三角膜、二面角弯曲、四面体、弹簧——种类多了，但装配的节奏一模一样：先放惯性块，再遍历邻接元件逐项累加。这一节我们盯住惯性块和最后那个 $3\times3$ 解；各种弹性元件本身怎么算，留到 Part 9 展开。

从 TinyVBD 看 一个 kernel = 一遍 solve() 的循环体

TinyVBD 的 solve() 用一个 C++ for (iV …) 串行扫顶点；Newton 把循环体提出来变成一个 kernel 函数，由 wp.launch 一次性铺到一整批顶点上并行跑。iV 这个循环变量，在 kernel 里就是开头那行 t_id = wp.tid() 取到的 thread 索引。你读 solve_elasticity 时，把它当成「solve() 的 for 循环里那一坨，被一个个 GPU thread 同时执行」就对了。

惯性项：和 solve() 逐字符对上

kernel 一开头，过滤掉非 active、零质量的顶点后，立刻装配惯性块。这是从真实代码摘出来的短切片（弹性元件的累加省略掉，那是 Part 10 的事）：

particle_vbd_kernels.py
@wp.kernel
def solve_elasticity(dt: float, particle_ids_in_color: wp.array[wp.int32],
                     pos_prev, pos, mass, inertia, particle_flags,
                     # ... tri / edge / tet / material arrays, adjacency ...
                     particle_forces, particle_hessians,
                     particle_displacements: wp.array[wp.vec3]):
    t_id = wp.tid()
    particle_index = particle_ids_in_color[t_id]

    if not particle_flags[particle_index] & ParticleFlags.ACTIVE or mass[particle_index] == 0:
        particle_displacements[particle_index] = wp.vec3(0.0)
        return

    dt_sqr_reciprocal = 1.0 / (dt * dt)

    # inertia force and hessian
    f = mass[particle_index] * (inertia[particle_index] - pos[particle_index]) * (dt_sqr_reciprocal)
    h = mass[particle_index] * dt_sqr_reciprocal * wp.identity(n=3, dtype=float)

    # ... elastic + contact force/Hessian accumulated into f, h ...

    h = h + particle_hessians[particle_index]
    f = f + particle_forces[particle_index]

    if abs(wp.determinant(h)) > 1e-8:
        h_inv = wp.inverse(h)
        particle_displacements[particle_index] = particle_displacements[particle_index] + h_inv * f

盯住那两行 f = … 和 h = …。对照公式：

$$\mathbf f \mathrel{+}= \frac{m_i}{h^2}\big(\mathbf y_i - \mathbf x_i\big),\qquad \mathbf H \mathrel{+}= \frac{m_i}{h^2}\,\mathbf I_3$$

mass[particle_index] 是 $m_i$，inertia[particle_index] 是惯性目标 $\mathbf y_i$（由 8.3 的 forward_step 用朴素惯性 $\mathbf y=\mathbf x+(\mathbf v+\mathbf g\,h)\,h$ 算出，没有 TinyVBD 那套自适应初始化），pos[particle_index] 是当前 $\mathbf x_i$，dt_sqr_reciprocal 是 $1/h^2$。力是负梯度，所以是 (inertia - pos) 的方向——把顶点往惯性碗底 $\mathbf y_i$ 拉。Hessian 的惯性块是各向同性的 $\frac{m_i}{h^2}\mathbf I_3$，恒正定，是后面那个 $3\times3$ 能解出来的定海神针。把这两行和 4.3 的 TinyVBD 惯性块并排看，除了 wp.identity(n=3) 换掉了 Mat3::Identity()、变量名带了下标，几乎一字不差。

每个邻接元件加一块 f / H

惯性块之后，被我用 # ... elastic + contact force/Hessian ... 省掉的那一大段，做的事和 solve() 内层 for (iNeiE …) 循环结构相同：遍历这个顶点的邻接元件，每个元件调一个 @wp.func 求出它对 $\mathbf f$、$\mathbf H$ 的贡献，然后 f = f + f_elem、h = h + h_elem 累加上去。Newton 里的元件种类是：

  • membrane（cloth 三角面，Neo-Hookean 膜能量）——对应 tri_indices 那段邻接面循环；
  • bending（二面角弯曲，dihedral angle）——对应 edge_indices 那段邻接边循环；
  • tet（四面体软体的 volumetric Neo-Hookean）——对应 tet_indices 那段邻接四面体循环；
  • spring（弹簧，力/Hessian 形式与 TinyVBD 弹簧完全一致）——以及 self-contact / body-particle contact，这些力/Hessian 在 sweep 里先攒进 particle_forces / particle_hessians，kernel 末尾两行 h = h + particle_hessians[…]、f = f + particle_forces[…] 把它们合并进来。

每种元件具体怎么算（变形梯度 $\mathbf F$、能量密度、SPD 投影……）是 Part 9 的主题，这里只要记住：它们都只是往同一个 $\mathbf f$、$\mathbf H$ 上 += 一块。这正是 3.3 求和式 $\sum_{j\in\mathcal F_i}$ 在 3D、多元件下的逐项实现。代码无脑累加、不在元件层面对总 $\mathbf H$ 做投影——和 TinyVBD 一个脾气，靠惯性块兜底。

interactive慢动作：逐顶点的 f_i / H_i 与 G 单调下降
这是串行版的直觉；下一节看 Newton 怎么按 color 并行做同样的事。

那个 3×3 解：Δx = H⁻¹f 与 det 守门

累加完，h 是完整的 $\mathbf H_i$、f 是完整的 $\mathbf f_i$，最后就是一步牛顿——求顶点该挪多少：

$$\Delta\mathbf x_i = \mathbf H_i^{-1}\,\mathbf f_i$$

但 Newton 在求逆前先守一道门：if abs(wp.determinant(h)) > 1e-8。只有当 $\lvert\det(\mathbf H_i)\rvert$ 超过这个阈值，才认为 $\mathbf H_i$ 足够「非退化」、求逆数值上靠谱，于是 h_inv = wp.inverse(h) 直接把 $3\times3$ 求逆，算出 $\mathbf H_i^{-1}\mathbf f_i$；否则什么都不做——这个顶点这一轮的位移保持原样（不加增量），等下一遍 sweep 它的邻居动过、$\mathbf H_i$ 重新变好再说。这就是 3.5 讲的「万一退化就跳过」在 GPU 上的具体写法。

还有个 GPU 细节值得点出：结果不是直接写回位置，而是累加进 per-vertex displacement——particle_displacements[particle_index] += h_inv * f。位移先攒着，等这一 color 的所有顶点都算完，再由外层（8.5）统一施加并做 penetration-free 截断。这样同色顶点之间互不干扰，才能安全并行。

从 TinyVBD 看 三行对三行

把 solve_elasticity 接回你吃透的 solve()，几乎是逐行对译：
① 惯性力 f = mass*(inertia - pos)/dt² ↔ TinyVBD 的 f = vertexMass(iV)*(inertia.col(iV) - vertex(iV))*dtSqrReciprocal，同一行，只是 3D。
② 邻接元件累加 h = h + h_elem / f = f + f_elem ↔ TinyVBD 内层循环里逐条边的 h += h_1_1 与带 ± 号的 f ±= …*diff；TinyVBD 的元件只有弹簧，Newton 多了膜/弯曲/tet，但都是往同一个 h、f 上 +=。
③ 解 h_inv = wp.inverse(h); dx = h_inv * f ↔ TinyVBD 的 dx = h.colPivHouseholderQr().solve(f)。同一个 $\Delta\mathbf x=\mathbf H^{-1}\mathbf f$——只是 Newton 对 $3\times3$ 直接求逆（加 det 守门），TinyVBD 走 Eigen 的列主元 QR 分解。两条路解的是同一个 $3\times3$ 系统。

直接求逆 vs colPivHouseholderQr

为什么 Newton 敢对 $\mathbf H_i$ 直接 wp.inverse，而 TinyVBD 用更稳的 colPivHouseholderQr？因为这是个 $3\times3$——闭式逆便宜又够用，GPU 上一个 thread 算它毫无压力；那道 abs(det) > 1e-8 的守门正是用来挡掉求逆会炸的退化情形。TinyVBD 在 CPU 上图省心用了 Eigen 现成的 QR，两者解的是同一个方程，数值上对良态的 $3\times3$ 结果一致。

代一组真数字：一个顶点的 3×3 装配全程

公式对完了，但 4.3 解剖 solve() 时缺了一步——真把数字代进去算一遍。这里补上：仿照 12-7 的节奏，给一个顶点摆出惯性块 + 一条具体弹簧，把 $\mathbf f_i$、$\mathbf H_i$ 拼出来，最后解出 $\Delta\mathbf x=\mathbf H_i^{-1}\mathbf f_i$，每个中间值都写出来让你能拿计算器复算。

设顶点 $i$ 当前在 $\mathbf x_i=(0,0,0)$，质量 $m=1$，时间步 $dt=\tfrac{1}{60}$，于是 $1/h^2=3600$。这一步的惯性目标 $\mathbf y_i$ 落在 $i$ 的正下方 $0.02$ 处（受重力下坠后的预测位置），即 $\mathbf y_i-\mathbf x_i=(0,-0.02,0)$。先填惯性块：

$$\mathbf f_{\text{inertia}}=\frac{m}{h^2}(\mathbf y_i-\mathbf x_i)=3600\times(0,-0.02,0)=(0,\,-72,\,0)$$
$$\mathbf H_{\text{inertia}}=\frac{m}{h^2}\mathbf I_3=3600\,\mathbf I_3=\begin{pmatrix}3600&0&0\\0&3600&0\\0&0&3600\end{pmatrix}$$

这就是 kernel 开头那两行 f = mass*(inertia-pos)/dt²、h = mass/dt²·I₃ 的数值版：力 $(0,-72,0)$ 竖直向下、把顶点往惯性碗底 $\mathbf y_i$ 拽；Hessian 是干干净净的 $3600\,\mathbf I_3$，三个方向各向同性、恒正定。

现在加一条具体弹簧。它把 $i$ 连到一个被钉住的邻居 $\mathbf x_{nb}=(1.2,0,0)$，刚度 $k=100$、rest 长度 $l_0=1.0$。算几何量（记号同 1.5，$\mathbf d=\mathbf x_i-\mathbf x_{nb}$）：

$$\mathbf d=\mathbf x_i-\mathbf x_{nb}=(-1.2,\,0,\,0),\qquad l=\lVert\mathbf d\rVert=1.2,\qquad \frac{l_0}{l}=\frac{1.0}{1.2}\approx0.833$$

弹簧被拉长了（$l=1.2>l_0=1.0$）。先算它对 $i$ 的力 $\mathbf f=k\frac{l_0-l}{l}\mathbf d$：

$$k\frac{l_0-l}{l}=100\times\frac{1.0-1.2}{1.2}=100\times(-0.1667)=-16.67,\qquad \mathbf f_{\text{spring}}=-16.67\times(-1.2,0,0)=(20,\,0,\,0)$$

力沿 $+x$、大小 $20$——被拉长的弹簧把 $i$ 往邻居方向拽回去，方向和符号都对。再算弹簧 Hessian $\mathbf H_{aa}=k\big[\mathbf I-\frac{l_0}{l}(\mathbf I-\frac{\mathbf d\mathbf d^\top}{l^2})\big]$。因为 $\mathbf d$ 只有 $x$ 分量，外积 $\mathbf d\mathbf d^\top/l^2$ 只有 $xx$ 项为 $1$，于是 $\mathbf I-\frac{\mathbf d\mathbf d^\top}{l^2}=\operatorname{diag}(0,1,1)$，整块是对角的：

$$\mathbf H_{\text{spring}}=100\Big[\mathbf I-0.833\,\operatorname{diag}(0,1,1)\Big]=100\,\operatorname{diag}(1,\,0.1667,\,0.1667)=\operatorname{diag}(100,\,16.67,\,16.67)$$

读它的物理：沿弹簧方向（$x$）特征值是 $k=100$（拉伸方向永远硬）；垂直方向是 $k(1-l_0/l)=100\times0.1667=16.67$，因为被拉长所以为正。（若改成压缩 $l<l_0$，这个垂直特征值就翻负——正是 1.5 那个不定来源，下一章 10.2 要处理的。）

惯性块 + 弹簧块累加，拼出完整的 $\mathbf f_i$、$\mathbf H_i$。两块 Hessian 都是对角的，加起来还是对角，$3\times3$ 解一眼就出：

$$\mathbf f_i=(0,-72,0)+(20,0,0)=(20,\,-72,\,0),\qquad \mathbf H_i=\operatorname{diag}(3700,\,3616.67,\,3616.67)$$
$$\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i=\Big(\frac{20}{3700},\;\frac{-72}{3616.67},\;0\Big)\approx(0.0054,\,-0.0199,\,0)$$

这一步把顶点往下挪约 $0.02$（追惯性目标）、往邻居方向挪约 $0.005$（弹簧拉回）。注意 $\det(\mathbf H_i)=3700\times3616.67^2\approx4.8\times10^{10}\gg10^{-8}$，远远过得了 kernel 里 abs(wp.determinant(h)) > 1e-8 那道守门——惯性块抬的 $3600$ 底，让 $\mathbf H_i$ 稳稳可逆。把这一遍和 4.3 的弹簧装配并排看：同一套力、同一套 Hessian、同一个 $\mathbf H^{-1}\mathbf f$，只是从 2D strand 升到 3D、从一行 C++ 变成一个 GPU thread。

这组数是为讲解构造的自洽示例

声明在前：这里的 $\mathbf y_i-\mathbf x_i=(0,-0.02,0)$、$k=100$、$l_0=1.0$ 等是为手算专门挑的干净自洽数，不是某次真实仿真的快照。能写准也值得记的是每一步怎么由上一步推出来、方向与符号怎么对——惯性力向下、拉长弹簧的力指回邻居、对角 Hessian 三个特征值各是什么。具体小数会随质量、刚度、单位整体缩放；重点是装配链路，不是某个数字。

这一节你要带走的

一句话记牢：Newton 的 solve_elasticity kernel 就是 TinyVBD solve() 的循环体搬上 GPU——每个 thread 一个顶点，先用 mass*(inertia-pos)/dt² 和 (mass/dt²)·I₃ 打底惯性块，再让每个邻接的 membrane / bending / tet / spring 元件各 += 一块 $\mathbf f$、$\mathbf H$，最后在 abs(det(H)) > 1e-8 守门下直接 wp.inverse(h) 求 $\Delta\mathbf x=\mathbf H^{-1}\mathbf f$，把结果累加进 per-vertex displacement。 公式还是 3.3 那个 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$，一个字没改。
