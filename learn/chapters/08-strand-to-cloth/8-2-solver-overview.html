SolverVBD 全景：step() 的三段式

上一节我们确认了「min G(x) 没变，变的是从一根 strand 串行变成一块 cloth 在 GPU 上按颜色并行」。现在我们站到 Newton 的 SolverVBD 顶层，把它的 step() 当一张地图看清楚——它就是 TinyVBD 那个 simulate() 每个 substep 的放大版。整个 step() 工整地分成 三段：Initialize（前向积分）、Iterate（迭代下降）、Finalize（由位移算速度）。这一节先把骨架立起来，后面几节再钻进每一段。

求解器契约：step() 是什么

Newton 里所有求解器都遵守同一个契约（7.3 讲过 Model / State 分离）：你把这一时刻的 State 交给求解器，它算出下一时刻的 State。签名长这样（solver_vbd.py:1578）：

solver_vbd.py · step()
def step(self, state_in, state_out, control, contacts, dt):
    ...

state_in 是输入快照、state_out 写输出、contacts 是 model.collide() 填好的碰撞结果、dt 是这一步的时间步。这正对应 TinyVBD 里 simulate() 每个 substep 调一次的那套流程——只不过 TinyVBD 把「输入」和「输出」都放在同一个 strand 对象里原地改，Newton 则显式用两个 State 做 ping-pong（7.3 的 state_in, state_out = state_out, state_in）。

关键事实先摆明：SolverVBD 对 particle 用 VBD、对 rigid body 用 AVBD（solver_vbd.py:88 的类 docstring）。本书的主线是 particle，所以下面凡是 rigid 的部分我们只点名它存在、不展开（rigid 的 AVBD penalty 机制留到 10.1）。step() 的 docstring 自己就把结构写得清清楚楚（solver_vbd.py:1588）：「The solver follows a 3-phase structure」。

三段式骨架

把 rigid / particle 两条线并排，step() 的主体就是下面这个骨架（solver_vbd.py:1613 起）：

solver_vbd.py · step() 主体（节选骨架）
# —— Phase 1: Initialize ——
self._initialize_rigid_bodies(state_in, control, contacts, dt, update_rigid)
self._initialize_particles(state_in, state_out, dt)

# —— Phase 2: Iterate ——
for iter_num in range(self.iterations):
    self._solve_rigid_body_iteration(state_in, state_out, control, contacts, dt)
    self._solve_particle_iteration(state_in, state_out, contacts, dt, iter_num)

# —— Phase 3: Finalize ——
self._finalize_rigid_bodies(state_in, state_out, dt, ...)
self._finalize_particles(state_out, dt)

就这么三块。每一段下面 rigid 和 particle 各有一个对应的方法，被并排调用。下面逐段说 particle 那一支在干什么。

Phase 1 · Initialize

_initialize_particles（solver_vbd.py:1734）做两件事：前向积分把每个顶点推到惯性预测位置 $\mathbf y$，再做一次 penetration-free 截断保证起点不穿透。前向积分就是你在 1.2 吃透的惯性位置：

$$\mathbf y = \mathbf x^t + h\,\mathbf v^t + h^2\,\mathbf a_{ext}$$

这一步也对应 TinyVBD 的 forwardStep()。但这里藏着本篇第一个要诚实点名的差异：Newton 用的是朴素惯性初始化（直接拿 $\mathbf y$ 当初值），不是 TinyVBD 那套带 accelerationComponent 的自适应初始化。这个对照是下一节 8.3 的主菜，这里先埋个伏笔。_initialize_rigid_bodies（solver_vbd.py:1771）则是给 rigid 的 AVBD 做 warm-start（恢复上一帧的 penalty / lambda），与 particle 无关，知道它存在即可。

Phase 2 · Iterate

这是下降发生的地方，也是整个 VBD 的心脏：for iter_num in range(self.iterations)。循环体里 rigid 和 particle 交替推进——先 _solve_rigid_body_iteration（AVBD 一轮），再 _solve_particle_iteration（VBD 一轮，solver_vbd.py:2154）。每一轮 particle iteration 不是一次简单 sweep，而是按颜色扫一遍所有 color group：

$$\texttt{for color in particle\_color\_groups:}\quad \text{同色顶点并行解 } \Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$$

同一个 color 里的顶点彼此不共享 force element，可以在 GPU 上并行更新；不同 color 之间串行，于是整体仍是 Gauss-Seidel 的收敛行为（8.5 专讲着色）。这正是把 TinyVBD solve() 那个单串行 sweep「按颜色切片、每片并行」的结果——min G(x) 一点没变，每个顶点解的还是那个 3×3 牛顿步，只是惯性 / 弹性 / 接触各项现在是 3D 的、且来自多种 force element（membrane / bending / spring / contact，8.4 拆）。

额外要点明：particle 这一支的迭代里没有 Chebyshev 加速。TinyVBD 有 applyAccelerator（4.4），但 Newton 的 particle 路径只有按颜色的 Gauss-Seidel，没有那一层 semi-iterative 加速——这是本篇第二个诚实对照点。

Phase 3 · Finalize

迭代收敛后，_finalize_particles（solver_vbd.py:2766）把位移换算成速度——后向差分，和 TinyVBD 的 updateVelocity() 一模一样：

$$\mathbf v^{t+1} = \frac{\mathbf x^{t+1}-\mathbf x^t}{h}$$

VBD 求解器全程只动位置，速度从来不是被直接积分出来的，而是「位置变了多少 ÷ dt」反推出来的。_finalize_rigid_bodies（solver_vbd.py:2779）对 rigid 做同类收尾（还含 Dahl friction 的持久状态更新），同样属 rigid 支，略过。三段走完，state_out 就是下一时刻的完整 State。

substeps vs iterations：两层循环别搞混

这里有两个容易混的「次数」，必须分清——它们在两个不同的循环层级上：

名字是什么在哪一层
substeps把一帧的 dt 切成几个更小的时间步，每个 substep 完整跑一次 step()（含三段）主循环里，外层：每个 substep 用 dt/substeps 各调一次 step()
iterations一次 step() 内 Phase 2 迭代下降的轮数（self.iterations）step() 内部，内层：for iter_num in range(self.iterations)

用一句话记牢层级：一帧 → 若干 substep → 每个 substep 一次 step() → step() 里 iterations 轮迭代 → 每轮按颜色扫。substeps 是把时间切细（改善大变形 / 高刚度的稳定与精度），iterations 是把同一个 $\arg\min G$ 解得更准。这和 TinyVBD 完全同构：那边 simulate() 先 dt /= substeps，每帧每 substep 跑一次「forwardStep → numIterations 次 solve → updateVelocity」（4.5）。Newton 的 self.iterations 就是 TinyVBD 的 numIterations。

从 TinyVBD 看 step() 的三段，就是 simulate() 每 substep 的三步

把 Newton 的 step() 和 TinyVBD simulate() 的每个 substep 一一对上，你会发现是同一套骨架穿了不同的衣服：

  • Phase 1 Initialize（_initialize_particles，前向积分到 $\mathbf y$）↔ TinyVBD 的 forwardStep()。
  • Phase 2 Iterate（for iter_num in range(self.iterations)，每轮按颜色解 $\Delta\mathbf x=\mathbf H^{-1}\mathbf f$）↔ TinyVBD 的 numIterations × solve() 循环。
  • Phase 3 Finalize（_finalize_particles，由位移算速度）↔ TinyVBD 的 updateVelocity() + saveOutputs()。

区别只在 Phase 2 内部那一层：TinyVBD 一次 solve() 是一遍串行扫完所有顶点；Newton 一轮 iteration 是逐 color 并行扫完所有顶点。最外的三段、最外的「迭代 N 轮」结构，分毫不差。

这一节你要带走的

记住这张地图：SolverVBD.step() = Initialize（前向积分到惯性位置 $\mathbf y$ + penetration-free 截断）→ Iterate（iterations 轮，每轮 rigid AVBD 与 particle VBD 交替，particle 那轮按 color 并行解 3×3 牛顿步）→ Finalize（由位移后向差分算速度）。它是 TinyVBD simulate() 每个 substep 的放大版：forwardStep↔Initialize、numIterations × solve()↔Iterate、updateVelocity↔Finalize。两处差异已埋伏：particle 用朴素惯性初始化（非自适应）、particle 路径无 Chebyshev。还分清了 substeps（外层切时间）与 iterations（内层解同一个 min G）。
