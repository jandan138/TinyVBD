弹性元件谱

在 TinyVBD 里，1.5 那根 strand 只认识一种 force element：弹簧。可一块真实的布、一团软体，要的远不止这个。这一节把 Newton particle VBD 支持的整套弹性元件铺开来看——Neo-Hookean membrane、volumetric Neo-Hookean、dihedral bending、spring——并点破一件事：它们各自只是往 8.4 那个 per-vertex 装配里，多塞一个 $\mathbf f$ 块和一个 $\mathbf H$ 块而已。讲完，我们把一块真在跑 VBD 的布请上台。

同一个装配口，多种元件往里叠

先把 8.4 的结论钉死：每个顶点 $i$ 解的还是那一步牛顿

$$\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i,\qquad \mathbf f_i=\frac{m_i}{h^2}(\mathbf y_i-\mathbf x_i)-\sum_{j\in\mathcal F_i}\frac{\partial E_j}{\partial\mathbf x_i},\qquad \mathbf H_i=\frac{m_i}{h^2}\mathbf I+\sum_{j\in\mathcal F_i}\frac{\partial^2 E_j}{\partial\mathbf x_i^2}.$$

惯性项那块谁都一样。变的只是求和里那一项项 $E_j$——它来自哪种 force element。strand 上 $\mathcal F_i$ 全是弹簧；一块布上，$\mathcal F_i$ 可能同时含三角形膜、二面角弯曲、几根加强弹簧。每种元件贡献一个 $3\times3$ 的力块与 Hessian 块，累加进同一个 $\mathbf f_i$ / $\mathbf H_i$。VBD 的好处正在这里：装配口是统一的，加一种材料就是多写一个 evaluate_*_force_hessian 的 @wp.func，主循环一行不用动。

Neo-Hookean membrane：布的面内能量

一块布最核心的元件是三角形膜。它管的是面内的拉伸与剪切：把三角形拉大、压扁、斜剪，都要花能量。和弹簧不同，它的能量不写在「边长」上，而写在 deformation gradient $\mathbf F$ 上——把 rest 三角形映到当前三角形的那个线性变换。布是个二维面嵌在三维里，所以 $\mathbf F$ 是个 $3\times2$ 矩阵（两列 $\mathbf f_0,\mathbf f_1$）。

Newton 用的是 stable Neo-Hookean 能量（Smith et al. 2018 改写到 shell 上）：

$$\psi=\frac{\mu}{2}(I_c-2)+\frac{\lambda}{2}(J_s-\alpha)^2,\qquad I_c=\operatorname{tr}(\mathbf F^\top\mathbf F),\quad J_s=\sqrt{\det(\mathbf F^\top\mathbf F)},\quad \alpha=1+\frac{\mu}{\lambda}.$$

直觉拆开看：$I_c$ 是两列的长度平方之和，惩罚「整体被拉伸」；$J_s$ 是当前三角形与 rest 三角形的面积比，$(J_s-\alpha)^2$ 惩罚「面积变化」。那个 $\alpha=1+\mu/\lambda$ 不是随手写的——它让 rest 状态（$\mathbf F$ 是等距映射）恰好应力为零，省掉「松弛后还有残余张力」的毛病。代码里还要做一道 Lamé 参数转换 $\mu_{NH}=\mu$、$\lambda_{NH}=\lambda+\mu$（Smith 2018 §3.4），这是 stable NH 的标准约定。

为什么 Hessian 要 SPD 投影

和压缩弹簧一样（1.5 那个变负的 $\lambda_\perp$），膜在被压缩时局部 Hessian 也会出现负特征值。Newton 在这里做 SPD projection：用 s_clamp = max(0, s) 把惹事的那一项的系数夹到非负，保证喂给牛顿步的 $\mathbf H_i$ 正定、$\Delta\mathbf x_i$ 方向可靠。注意这正是 TinyVBD 没做的——strand 靠惯性偏置和行列式阈值兜底，cloth 维度更高、更刚，索性把投影做实。下一节 10.2 专讲它。

下面是这块膜元件的核心几行（已删去 damping 与组装样板），看它如何从 $\mathbf F$ 一路算到力与 Hessian：

particle_vbd_kernels.py
# Deformation gradient F = [f0, f1]  (3x2，两列向量)
f0 = x01 * DmInv00 + x02 * DmInv10
f1 = x01 * DmInv01 + x02 * DmInv11

# J_s = 面积比 = sqrt(det(F^T F))
J_s = wp.sqrt(wp.max(f0_dot_f0 * f1_dot_f1 - f0_dot_f1 * f0_dot_f1, 1.0e-20))

# stable Neo-Hookean 参数 (Smith 2018, §3.4): mu_NH = mu, lambda_NH = lambda + mu
mu_nh  = mu
lmbd_nh = lmbd + mu
alpha  = 1.0 + mu_nh / lmbd_safe

# First Piola-Kirchhoff stress: P = mu*F + lambda*(J_s - alpha)*[g0, g1]
s = lmbd_nh * (J_s - alpha)
P_col0 = mu_nh * f0 + s * g0
P_col1 = mu_nh * f1 + s * g1
force = -(P_col0 * df0_dx + P_col1 * df1_dx)

# Hessian (per-vertex 3x3, SPD-projected): max(0, s) 是紧的 PSD 夹取
s_clamp = wp.max(0.0, s)
hessian = I_coeff * I33 + c1 * wp.outer(dJ_dx, dJ_dx) - r * wp.outer(w, w)

代一遍：一个均匀拉伸 10% 的三角形

公式列完了，代个数走一遍 $\mathbf F\to I_c\to J_s\to\alpha\to\psi$ 才踏实。设这块三角形被均匀拉伸 10%——变形梯度的两列各拉长到 $1.1$、互相正交：

$$\mathbf f_0=(1.1,0,0),\quad \mathbf f_1=(0,1.1,0)\ \Rightarrow\ \mathbf F^\top\mathbf F=\begin{pmatrix}1.21&0\\0&1.21\end{pmatrix}$$

由它一路算下去（每个中间值都能用计算器复算）：

$$I_c=\operatorname{tr}(\mathbf F^\top\mathbf F)=1.21+1.21=2.42,\qquad J_s=\sqrt{\det(\mathbf F^\top\mathbf F)}=\sqrt{1.21\times1.21}=1.21$$

$J_s=1.21$ 就是当前三角形比 rest 大了 $21\%$ 的面积比（$1.1\times1.1$）。取一组示意 Lamé 参数 $\mu=1,\ \lambda=10$，则 $\alpha=1+\mu/\lambda=1.1$，代进能量：

$$\psi=\frac{\mu}{2}(I_c-2)+\frac{\lambda}{2}(J_s-\alpha)^2=\frac{1}{2}(2.42-2)+\frac{10}{2}(1.21-1.1)^2$$
$$=0.5\times0.42+5\times(0.11)^2=0.21+0.0605\approx0.27$$

读出来：两项都为正、都在「储能」——拉伸项 $\frac{\mu}{2}(I_c-2)=0.21$ 罚「两列被拉长」，面积项 $\frac{\lambda}{2}(J_s-\alpha)^2=0.06$ 罚「面积偏离 $\alpha$」。把三角形松回 rest（$\mathbf F$ 为等距、$I_c=2$、$J_s=1$），第一项归零；面积项因为 $\alpha=1.1\neq1$ 不会同时归零，但梯度在 rest 处恰好为零（这正是 $\alpha=1+\mu/\lambda$ 的作用——rest 无应力）。压缩时（$J_s<\alpha$）面积项的系数 $s=\lambda(J_s-\alpha)$ 翻负、Hessian 趋于不定，于是代码里那道 s_clamp = max(0, s) 上场——下一节 10.2 专讲。

这些数是自洽示例，量级随材料/单位变

$\psi\approx0.27$、$\mu=1,\lambda=10$ 这组是为讲解构造的自洽示例，不是某种真实材料的标定值。能量的绝对量级会随 Lamé 参数、网格尺度、单位整体缩放（换一组 $\mu,\lambda$ 数字立刻变）。能写准也值得记的是链路和趋势：$\mathbf F\to I_c,J_s\to\psi$ 每步怎么推、两项各罚什么、压缩时 $s$ 怎么翻负。下面 dihedral 的 $\cos\theta$ 同理——几何关系是真的，具体角度随构型变。

误区：膜元件 ≠ 三根边弹簧

很容易把三角形膜想成「三条边各挂一根弹簧」——错，而且错得有后果。边弹簧只惩罚三条边的长度，对「面积」「剪切」是瞎的：一个三角形可以三条边长都不变，却被斜剪成扁平的一片（剪切形变），三根边弹簧对此零反应；它也压不住面积塌缩。膜元件写在变形梯度 $\mathbf F$ 上——$I_c$ 抓两列的拉伸、$J_s$ 抓面积比，剪切与面积变化都进了能量。直觉验证：上面那个例子里，就算我把三角形剪成 $\mathbf f_0\cdot\mathbf f_1\neq0$（两列不再正交），$\det(\mathbf F^\top\mathbf F)$ 立刻改变、$J_s$ 跟着变、$\psi$ 报警；三根只认边长的弹簧却可能一点没察觉。所以布要绷得住、剪得动有回弹，必须用膜，不能拿边弹簧凑。

Volumetric Neo-Hookean：软体的体积能量

把布换成一块软体（橡皮、肌肉、果冻），网格从三角形变成四面体 tet。此时 $\mathbf F$ 是堂堂正正的 $3\times3$ 矩阵，能量也升级成体积版 Neo-Hookean：

$$\psi=\frac{\mu}{2}\,(I_1-3-2\ln J)+\frac{\lambda}{2}\,(J-1)^2,\qquad I_1=\operatorname{tr}(\mathbf F^\top\mathbf F),\quad J=\det\mathbf F.$$

读法和膜版同构：$I_1$ 项管形变（剪切+拉伸），$(J-1)^2$ 项管体积变化，$J$ 是体积比。那个 $-2\ln J$ 是 Neo-Hookean 的招牌——当 $J\to0$（体积被压到塌缩），$\ln J\to-\infty$，能量爆炸，于是材料天然抗拒翻面/塌陷，这是线性弹性给不了的。元件的角色完全不变：算出力块与 Hessian 块，叠进顶点的 $\mathbf f_i/\mathbf H_i$。从 strand 的弹簧到 tet 的体积能量，变的只是「$E_j$ 长什么样」，装配机理一字未改。

Dihedral bending：让布不愿意折

光有膜，布会像保鲜膜一样软塌塌，没有「抗弯」。dihedral bending 元件补上这点：它盯着共享一条内部边的两个三角形之间的二面角 $\theta$，谁想把布折起来就向谁要能量：

$$E=\frac{k\,L_{\text{rest}}}{2}\,(\theta-\theta_{\text{rest}})^2.$$

$\theta_{\text{rest}}$ 是 rest 折角（平的布是 $0$），$L_{\text{rest}}$ 把刚度按边长缩放。一个干净的工程细节：边界边被跳过——一条没有「对面三角形」的边压根构不成二面角。代码开头那个 == -1 判断就是干这个的，对应 FACTS 里「boundary edges skipped」。整段从两个三角法向算 $\theta$、再写 $\dfrac{\partial E}{\partial\theta}=k\,L_{\text{rest}}(\theta-\theta_{\text{rest}})$，结构上和弹簧的「先标量能量、再链式法则铺到顶点」一模一样，只是几何量从「边长」换成了「夹角」。

那 $\theta$ 从哪来？前面只说「从两个三角法向算」，这里把它补全——这是全节唯一一处之前真正略过的几何。共享内部边的两个三角形各算一个面法向 $\mathbf n_1,\mathbf n_2$（边的两侧各一个三角形叉积得到），二面角就藏在这两个法向的夹角里：

$$\cos\theta=\frac{\mathbf n_1\cdot\mathbf n_2}{\lVert\mathbf n_1\rVert\,\lVert\mathbf n_2\rVert}$$

读法很直观：布是平的时候，两个三角形共面、法向同向，$\mathbf n_1\cdot\mathbf n_2$ 取最大，$\cos\theta=1$ → $\theta=0$（rest，不发力）；沿这条边把布对折，两个法向逐渐张开，$\cos\theta$ 下降，$\theta$ 增大，$(\theta-\theta_{\text{rest}})^2$ 拉起能量、回复力把布往平里掰。$\lVert\mathbf n_1\rVert\lVert\mathbf n_2\rVert$ 的归一化是必须的——否则三角形面积一大法向就长，会污染纯粹的「角度」信号。代码用这个 $\cos\theta$（必要时配叉积定符号区分凸折/凹折）还原出 $\theta$，再喂给上面的标量能量与 $\dfrac{\partial E}{\partial\theta}$，最后链式法则把力铺到边两端 + 两个对顶点共 4 个顶点上。

从 TinyVBD 看 spring 元件同形，cloth 只是多叠了几种

你已经把 TinyVBD 那根 strand 的单弹簧 Hessian 吃透了：$\mathbf H_{aa}=k\big[\mathbf I-\tfrac{l_0}{l}\big(\mathbf I-\tfrac{\mathbf d\mathbf d^\top}{l^2}\big)\big]$，力 $\mathbf f=k\tfrac{l_0-l}{l}\mathbf d$。Newton 的 spring 元件与它同形（用 spring_rest_length / stiffness / damping，力与 Hessian 是同一套公式）。真正的区别只有一句话：cloth 不过是在这根弹簧之外，又往同一个 $\mathbf f_i/\mathbf H_i$ 里叠加了 membrane、bending、（软体时还有）tet 几种 force element。装配口没变，下降原理没变，变的只是邻接元件的种类。你那根 strand 放大成一块布，就是这么发生的。

玩这块布时，盯住三件事

这个 cloth-lab 是整个 Newton 篇的压轴——你前面学的每一块，现在都在它里面同时转着。玩的时候建议有意识地看三处：

  • stiffness：调高，布更绷、抗拉；调低，布更软、垂得更狠。这是膜元件 $\mu,\lambda$ 在起作用。
  • bending：调高，布抗折、会保持平整甚至打卷；调到零，布软塌得像液面。这是二面角元件在干活。
  • show colors：切开看顶点的 graph coloring 分组——同色顶点不共享 force element，可在 GPU 上并行更新（8.5）。把迭代数压很低，它依旧不炸——这还是 VBD 那块招牌：单调下降、无条件稳定，只是从一根 strand 长成了一块布。

这一节你要带走的

三句话：(1) Newton particle VBD 是一谱弹性元件——membrane（布，$\mathbf F$ 为 $3\times2$）、volumetric NH（软体，$\mathbf F$ 为 $3\times3$）、dihedral bending（抗折）、spring（与 TinyVBD 同形）；(2) 每种元件只贡献一个力块与 Hessian 块，叠进 8.4 那个统一的 per-vertex 装配，主循环不变；(3) 高维更刚的膜/体积能量在压缩下 Hessian 会不定，于是 Newton 在这里做实了 SPD projection——这是 strand 没做、cloth 必须做的一步。

那道 SPD 投影到底怎么把负特征值拍正、又怎么不破坏物理，还有让布「甩两下就停」的 Rayleigh damping 是怎么挂进力与 Hessian 的——下一节 10.2 把这两件事讲透。
