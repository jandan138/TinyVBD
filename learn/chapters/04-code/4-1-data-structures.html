数据结构：Types.h 与 Strand.h

读代码先读数据。在我们逐行解剖 forwardStep() / solve() 之前，先把 TinyVBD 摆弄的两样东西看清楚：一套薄薄的 Eigen typedef（Types.h），和一根 strand 的全部状态（Strand.h）。把字段和前面 Part 1/Part 3 的公式对上号，后面读算法就是「按图索骥」。

Types.h：一层薄薄的 Eigen 别名

Types.h 几乎没有逻辑，它只做三件事：选一个浮点类型、给 Eigen 矩阵起一堆短名字、定义几个宏。先看浮点与维度宏：

Types.h
#define GRAVITY_AXIS 1

#define POINT_VEC_DIMS 3
#define F_FLATTEN_DIMS 9
#define X_FLATTEN_DIMS 12

#define SQR(x) ((x) * (x))
#define CUBE(x) ((x) * (x) * (x))

GRAVITY_AXIS 1 就是说重力沿 y 轴（轴 0/1/2 = x/y/z）。这和 main.cpp 里 gravity << 0, -10, 0 一致——重力指向 $-y$，模长 10。SQR 是个再普通不过的平方宏，但记住它：4.4 的 Chebyshev omega 递推里 SQR(pho) 就是 $\rho^2$。

核心是这一组 typedef（都在 namespace VBD 内）：

Types.h
typedef float FloatingType;
typedef const float CFloatingType;
typedef int32_t IdType;

typedef Eigen::Matrix<FloatingType, 3, 1> Vec3;
typedef Eigen::Matrix<IdType, 2, 1> Vec2I;
typedef Eigen::Matrix<FloatingType, POINT_VEC_DIMS, POINT_VEC_DIMS> Mat3;
typedef Eigen::Matrix<FloatingType, POINT_VEC_DIMS, Eigen::Dynamic> TVerticesMat;
typedef Eigen::Matrix<FloatingType, Eigen::Dynamic, 1> VecDynamic;

using Vec3Block = Eigen::Block<TVerticesMat, 3, 1>;

三个名字会在全书反复出现，务必记牢：

  • FloatingType = float：整套仿真跑在 单精度上（源码里 double 那行被注释掉了）。后面看到 1e8 的 stiffness 时，心里要清楚精度只有 float。
  • TVerticesMat = Matrix<float, 3, Dynamic>：3×N 的顶点矩阵，每一列是一个顶点的 $(x,y,z)$。strand 的位置、速度、惯性预测，全是这个形状。它是列主序的，「取第 i 个顶点」= 取第 i 列。
  • Vec3 / Mat3：单个顶点的 3 维向量、单个顶点的 3×3 矩阵。4.3 里每个顶点的力 f 是 Vec3，Hessian h 是 Mat3——这正是 VBD「一次只动一个顶点的 3 个自由度」在类型上的体现。

Vec2I 是一条边的两个整型顶点 id；VecDynamic 是动态长度的 float 向量（用来存每个顶点的质量、每条边的 rest 长度）。

RELEASE_ASSERT：失败就停下来给你看

Types.h 还定义了一个 RELEASE_ASSERT(expr) 宏（配 AbstractAsserter）：表达式为假时打印 expr / 文件 / 行号，执行一个可选的 debug 回调，getchar() 等你看一眼，然后 exit(-1)。它和标准 assert 的区别是 Release 构建下也生效——这是研究型代码常见的自保手段。TinyVBD 的热路径里其实主要用普通 assert（如 from() 里的尺寸检查），但这个宏放在工具箱里随时可用。

Mesh 基类：一根 strand 的最小状态

Strand.h 里先有一个极简的 Mesh 基类，它持有任何「一堆带质量的顶点」都需要的四件东西：

Strand.h
struct Mesh
{
    TVerticesMat mVertPos;
    TVerticesMat mVertPrevPos;
    TVerticesMat mVelocity;
    VecDynamic vertexMass;

    Vec3Block vertex(int i) {
        return mVertPos.block<3, 1>(0, i);
    }
};

mVertPos 是当前位置 $\mathbf x$，mVertPrevPos 是这一步开始前的位置 $\mathbf x^t$（用来在 4.5 的 updateVelocity() 里反算速度），mVelocity 是 $\mathbf v$，vertexMass 是每个顶点的质量 $m_i$。

关键是 vertex(i)：它返回 mVertPos 的第 i 列的一个 3×1 block 引用（不是拷贝）。所以 strand.vertex(iV) += dx; 是原地把第 iV 个顶点挪动 dx——这正是 4.3 里 Gauss-Seidel 每步落地的那一行。「返回引用、就地写回」是 Gauss-Seidel（而非 Jacobi）的关键：下一个顶点立刻看到上一个顶点的新位置。

VBDStrand：补上弹簧、邻接与历史

VBDStrand : public Mesh 在 Mesh 之上加了弹簧网络和算法需要的历史量：

Strand.h
struct VBDStrand : public Mesh
{
public:
    std::vector<Vec2I> edges;
    std::vector<FloatingType> edgesStiffness;
    std::vector<std::vector<int>> vertAdjacentEdges;
    size_t nEdges;
    FloatingType stiffness;
    FloatingType stiffnessSkipSpring;
    VecDynamic orgLengths;
    TVerticesMat mVelocitiesPrev;
    TVerticesMat inertia;
    size_t numVerts;

    TVerticesMat prevprevPos;
    bool hasVelocitiesPrev = false;
    bool hasApproxAcceleration = false;

逐个对上号：

  • edges：每条弹簧的两个端点 id（Vec2I）。edgesStiffness：每条边的刚度 $k$（注意是逐边存的，为高刚度比实验留了口子，见 5.2）。orgLengths：每条边的 rest 长度 $l_0$。
  • vertAdjacentEdges：每个顶点的邻接边 id 列表（一个 vector of vector）。这是全章最该记住的字段——4.3 里组装第 $i$ 个顶点的力 $\mathbf f_i$ 与 Hessian $\mathbf H_i$ 时，正是遍历 vertAdjacentEdges[iV] 把每条邻接弹簧的贡献累加上去。它就是 3.3 里那个「force elements 集合 $\mathcal F_i$」的代码化身。
  • inertia：惯性预测位置 $\mathbf y$（4.2 里算出来）。mVelocitiesPrev：上一帧速度 $\mathbf v^{t-1}$，hasVelocitiesPrev 标记它是否已经有了——服务于自适应初始化（4.2）。
  • prevprevPos：迭代历史 $\mathbf x^{(n-2)}$，只给 Chebyshev 加速用（4.4）。

两个 from(...)：怎么把一根 strand 装起来

VBDStrand 有两个 from(...) 重载，都从「顶点位置 + 质量 + 边长 + 刚度」构建出上面这些字段，区别只在刚度怎么给。先看统一刚度那个（带可选 skip spring）的核心循环：

Strand.h · from（统一刚度）
vertAdjacentEdges.resize( pos.size());
std::vector<FloatingType> lengthesNew;
for (size_t i = 0; i < lengthes.size(); ++i) {
    Vec2I edge;
    edge << i, i + 1;
    int edgeId = edges.size();
    edges.push_back(edge);

    vertAdjacentEdges[i].push_back(edgeId);
    vertAdjacentEdges[i+1].push_back(edgeId);
    edgesStiffness.push_back(stiffness);
    lengthesNew.push_back(lengthes[i]);
    if (addSkipSpring && i < lengthes.size() - 1)
    {
        edgeId = edges.size();
        edge << i, i + 2;
        edges.push_back(edge);
        vertAdjacentEdges[i].push_back(edgeId);
        vertAdjacentEdges[i + 2].push_back(edgeId);
        edgesStiffness.push_back(stiffnessSkipSpring);
        lengthesNew.push_back(lengthes[i] + lengthes[i+1]);
    }
}

主循环每一圈做一条「结构弹簧」：连接相邻顶点 $i$ 与 $i{+}1$，rest 长度就是给定的 lengthes[i]，刚度统一是 stiffness。注意它同时把这条边登记到两个端点的邻接表里（vertAdjacentEdges[i] 和 [i+1]）——这就是为什么后面一个顶点能找到「我连着哪些弹簧」。

下半段是 skip spring：当 addSkipSpring 为真，额外连一条 $i \to i{+}2$ 的弹簧，刚度用单独的 stiffnessSkipSpring（默认 1e2），rest 长度取 lengthes[i] + lengthes[i+1]（两段之和）。一根纯靠相邻弹簧串起来的 strand 是没有抗弯能力的——它能像面条一样随便对折。跳一格的弹簧给它一个「保持伸直」的倾向，近似了 bending（弯曲）刚度。默认实验 initializeTilted() 就开了它（skipSpringStrength=100），细节见 5.3。

从 PBD 看 邻接表就是约束图，只不过这里按顶点存

在 PBD 里你也维护一张「约束清单」，挨个投影。VBD 的 vertAdjacentEdges 是同一张图的另一种切法：不是「对每条约束做点什么」，而是「对每个顶点，找出所有影响它的约束」。这个转置正是 block coordinate descent 的前提——只有先知道「谁影响我」，才能为单个顶点组出局部的 $\mathbf f_i$、$\mathbf H_i$（3.3）。

第二个 from(...) 重载把最后一个参数换成 std::vector<FloatingType> stiffness_in，直接 edgesStiffness = stiffness_in; 收下逐边刚度数组，循环里不再 push 统一刚度、也不造 skip spring。initializeStiffRatio() 用它造出交替的 1e4 / 1e8 刚度，专门去拷打 VBD 在高刚度比下的收敛（5.2）。

一个边角：第二个 from 里的 push_back(stiffness)

逐边刚度版的循环里有一行 edgesStiffness.push_back(stiffness);，但 edgesStiffness 在循环前已被整段赋成 stiffness_in 了，所以这次 push 实际把数组又往后追加了一截（而 stiffness 成员此重载里没被设置）。这不影响结果，因为后续 solve() 只按 edgeId 索引前半段真正用到的刚度。提一句是为了你读源码时不被它绊住——真正决定刚度的是构造时传入的 stiffness_in。

这一节你要带走的

三件事：(1) TVerticesMat 是 3×N 的「每列一个顶点」矩阵，vertex(i) 返回可写的列引用——VBD「就地更新单个顶点」全靠它；(2) VBDStrand 把状态分成三类：弹簧网络（edges/edgesStiffness/orgLengths）、邻接（vertAdjacentEdges，组装 $\mathbf f_i/\mathbf H_i$ 的图）、历史（inertia/mVertPrevPos/mVelocitiesPrev/prevprevPos）；(3) from(...) 造出结构弹簧 $i\!-\!i{+}1$，可选的 skip spring $i\!-\!i{+}2$ 近似弯曲，逐边刚度版为高刚度比实验留口。
