Chebyshev 加速：omega 与 applyAccelerator

Gauss-Seidel 收敛是线性的——一遍遍 sweep 稳稳逼近，但有点慢。Chebyshev semi-iterative acceleration（Wang 2015）用一个几乎免费的小技巧把收敛曲线压陡：每次完整迭代后，沿着「这次和上上次之差」的方向外推一步。代码里就是 getAcceleratorOmega 加 applyAccelerator 两个函数。

直觉：往前一步多踩一脚

普通迭代每次只走到 $\bar{\mathbf x}^{(n)}$（这一遍 sweep 的输出）。Chebyshev 的想法：迭代的误差是一路朝同一方向缩的，那不如顺势多推一点。具体地，用一个随迭代变化的系数 $\omega_n$，把当前结果沿「与上上次 $\mathbf x^{(n-2)}$ 的差」放大：

$$\mathbf x^{(n)} = \omega_n\big(\bar{\mathbf x}^{(n)} - \mathbf x^{(n-2)}\big) + \mathbf x^{(n-2)}$$

$\omega_n>1$ 时这是外推（overshoot 一点点，抵消掉慢慢爬的尾巴）；$\omega_n=1$ 时就退回普通迭代。系数靠一条递推生成：

$$\omega_1 = 1,\quad \omega_2 = \frac{2}{2-\rho^2},\quad \omega_n = \frac{4}{4-\rho^2\,\omega_{n-1}}$$

这里 $\rho\in(0,1)$ 是迭代矩阵谱半径的估计值（estimated spectral radius）——它刻画「每遍迭代误差缩多少」。$\rho$ 越接近 1，意味着原始迭代越慢，需要越激进的外推；$\omega_n$ 随之冲得越高。论文里 $\rho$ 常取 $\approx 0.93\text{–}0.95$。

getAcceleratorOmega：那条递推

这就是上面递推的逐字实现：

main.cpp · getAcceleratorOmega()
FloatingType getAcceleratorOmega(int order, CFloatingType pho, CFloatingType prevOmega)
{
    switch (order)
    {
    case 1:
        return 1;
        break;
    case 2:
        return  2 / (2 - SQR(pho));
        break;
    default:
        assert(order > 0);

        return 4.f / (4 - SQR(pho) * prevOmega);;
        break;
    }
}

对照递推：order 是迭代序号 $n$，pho 是 $\rho$（变量名拼成了 pho），prevOmega 是 $\omega_{n-1}$。SQR(pho) 就是 4.1 那个平方宏算出的 $\rho^2$。三个分支正好是 $\omega_1=1$、$\omega_2=\frac{2}{2-\rho^2}$、$\omega_n=\frac{4}{4-\rho^2\omega_{n-1}}$。注意它需要上一次的 $\omega$ 喂回来——主循环里就是把上一轮的 omega 再传进来（见下）。

代数字：ρ=0.95 时 ω 头几项

递推式写出来了，但 $\omega$ 实际爬到多高、怎么稳定？取 $\rho=0.95$（$\rho^2=0.9025$）跑头几项，把它代成可复算的数（示例 $\rho=0.95$）：

$n$公式代入$\omega_n$
1$\omega_1=1$—$1$
2$\dfrac{2}{2-\rho^2}$$\dfrac{2}{2-0.9025}=\dfrac{2}{1.0975}$$\approx 1.822$
3$\dfrac{4}{4-\rho^2\omega_2}$$\dfrac{4}{4-0.9025\times1.822}=\dfrac{4}{4-1.645}$$\approx 1.698$
4$\dfrac{4}{4-\rho^2\omega_3}$$\dfrac{4}{4-0.9025\times1.698}=\dfrac{4}{4-1.533}$$\approx 1.621$

读这串数：$\omega$ 第二项一下冲到 $1.822$（外推最猛），随后回落、稳定收敛到一个 $>1$ 的定值（这里约 $1.6$ 上下）。$\omega>1$ 才外推、$\rho$ 越大冲得越高——这正好和上面 warn callout 的「推到 0.99 翻车」形成定量对照：$\rho=0.95$ 时 $\omega$ 收敛得很温和，再往 1 推就会把 $\omega$ 抬到外推过猛、开始振荡。

applyAccelerator：外推那一步

main.cpp · applyAccelerator()
void applyAccelerator(FloatingType omega) {
    if (omega > 1.f)
    {
        strand.mVertPos = omega * (strand.mVertPos - strand.prevprevPos) + strand.prevprevPos;
    }
}

这就是 $\mathbf x^{(n)} = \omega(\mathbf x - \mathbf x^{(n-2)}) + \mathbf x^{(n-2)}$ 的逐字实现：此刻 mVertPos 是刚 sweep 完的 $\bar{\mathbf x}^{(n)}$，prevprevPos 是 $\mathbf x^{(n-2)}$。只有 omega > 1 才动手（$\omega=1$ 是恒等变换，没必要算），所以前两次迭代（$\omega_1=1$，且 $\omega_2$ 在 $\rho$ 小时也 $\approx 1$）基本不外推，等递推把 $\omega$ 抬起来才真正发力。prevprevPos 这个字段（4.1 见过）就是专为它存在的，存的就是 $\mathbf x^{(n-2)}$。

simulate() 怎么把它接起来

回到 4.5 的迭代循环，加速逻辑是这样穿进去的：

main.cpp · simulate() 内层
float omega=1.f;
for (iter = 0; iter < params.numIterations; iter++)
{
    prevIterPos = strand.mVertPos;
    solve();

    if (params.useAcceleration)
    {
        omega = getAcceleratorOmega(iter + 1, params.accelerationRho, omega);
        applyAccelerator(omega);
        strand.prevprevPos = prevIterPos;
    }
}

读这个时序：每轮先把当前位置存进 prevIterPos，再 solve() 做一遍 Gauss-Seidel sweep。若开了加速，就 (1) 用 iter+1 当 order、把上轮 omega 喂回去算新 omega，(2) applyAccelerator 外推，(3) 把 prevprevPos 更新成这一轮 sweep 前的 prevIterPos。于是下一轮里的 prevprevPos 恰好是再上一轮的位置——也就是 $\mathbf x^{(n-2)}$，递推链就此闭合。

ρ 太大 → overshoot 振荡甚至发散

$\rho$ 是个估计值，调对了能把几十次迭代的活儿压到十几次；调过头就翻车。$\rho$ 越接近 1，$\omega_n$ 冲得越高、外推越猛——一旦超过实际谱半径，外推就从「抄近路」变成「冲过头再荡回来」，收敛曲线开始上下振荡，再大直接发散。这就是为什么它叫 semi-iterative「加速」而不是「保证」：它假设了误差谱的形状，假设错了要付代价。下面的部件让你亲手把 $\rho$ 推到 0.99 看它怎么抖。

诚实交代：TinyVBD 默认没开加速

别被这一章误导：默认配置里 useAcceleration = false，所以上面那段 if (params.useAcceleration) 整个不执行，getAcceleratorOmega / applyAccelerator 平时是死代码。默认的 accelerationRho = 0.5 也偏保守（论文常用 0.93–0.95）。默认实验 initializeTilted() 还显式写了 params.useAcceleration = false; params.accelerationRho = 0.5;。想试加速，去 main.cpp 把 useAcceleration 设为 true、把 accelerationRho 调到 0.9 上下再跑——这也是 5.4 实验菜单里的一道菜。

从 PBD 看 同一个加速器，也能套在别的迭代器上

Chebyshev semi-iterative 不是 VBD 专属——它是给任何线性收敛的不动点迭代提速的通用外壳（Wang 2015 最早就是套在 Projective Dynamics 的 local/global 上）。它不改你每步在算什么（这里还是 4.3 那遍 Gauss-Seidel sweep），只在外面加一层「沿历史方向外推」。代价是引入一个要手调的 $\rho$ 和翻车风险。所以 TinyVBD 把它做成可选开关、默认关掉——基线 VBD 不靠它也已经无条件稳定。

这一节你要带走的

三件事：(1) Chebyshev 加速 = 每遍 sweep 后做一次外推 $\mathbf x^{(n)}=\omega_n(\bar{\mathbf x}^{(n)}-\mathbf x^{(n-2)})+\mathbf x^{(n-2)}$，getAcceleratorOmega 跑递推 $\omega_1{=}1,\omega_2{=}\frac{2}{2-\rho^2},\omega_n{=}\frac{4}{4-\rho^2\omega_{n-1}}$，applyAccelerator 在 $\omega>1$ 时落地，prevprevPos 存 $\mathbf x^{(n-2)}$；(2) $\rho$ 是谱半径估计，越大越激进，太大则 overshoot 振荡/发散；(3) TinyVBD 默认 useAcceleration=false、accelerationRho=0.5，加速是关着的可选项。
