惯性项 y 与弹性能：一场拔河

上一节我们看到 $G(\mathbf x)$ 是惯性碗和弹性碗的拔河。这一节把这场拔河看到底：预测位置 $\mathbf y$ 究竟是什么、它就是代码里 forwardStep() 算的那一行；惯性系数 $m/h^2$ 如何决定碗的陡峭，从而决定谁赢；以及这场拔河的胜负如何牵动后面的高质量比实验和「好初值」。

先把那个能量摆回来

隐式欧拉一步 $=\arg\min_\mathbf{x} G(\mathbf x)$，而

$$G(\mathbf x)=\underbrace{\frac{1}{2h^2}\lVert\mathbf x-\mathbf y\rVert_M^2}_{\text{惯性项}}+\underbrace{E(\mathbf x)}_{\text{弹性能}}.$$

两项各是一个「碗」，最低点通常不重合，$\arg\min$ 落在二者妥协处。1.2 的 potential-landscape widget 已经让你亲手拖过这场拔河——把惯性权重调大，谷底被拉向 $\mathbf y$；把刚度调大，谷底被拽回 rest 环。这一节不再上新 widget，而是把那张图背后的两个旋钮——$\mathbf y$ 和系数 $m/h^2$——彻底讲清楚。

惯性项里的 y 到底是什么

$\mathbf y$ 叫predicted position（预测/惯性位置）：如果从这一刻起，把弹簧全拆了、只剩外力（重力）作用，质点纯靠惯性会漂到哪里。它的表达式是

$$\mathbf y=\mathbf x^t+h\,\mathbf v^t+h^2\,\mathbf a_{ext},$$

三项的意思一目了然：从当前位置 $\mathbf x^t$ 出发，沿当前速度 $\mathbf v^t$ 漂 $h$ 这么久，再加上重力等外部加速度 $\mathbf a_{ext}$ 在这段时间里攒下的位移 $h^2\mathbf a_{ext}$。这正是显式欧拉对「无内力质点」给出的下一步——也是这根头发「如果完全松掉」会去的地方。惯性项 $\frac{1}{2h^2}\lVert\mathbf x-\mathbf y\rVert_M^2$ 就是在惩罚「偏离这个自然漂移」，物理上等价于动量守恒的诉求。

从 GAMES103 看 y 就是你写过的「显式预测步」

GAMES103 写半隐式/隐式积分时，你十有八九先算过一个 $\mathbf x^*=\mathbf x+h\mathbf v+h^2\mathbf f_{ext}/m$ 当起点，再去修正。那个 $\mathbf x^*$ 和这里的 $\mathbf y$ 是同一个东西。区别只在视角：当年你把它当「迭代的初值」，现在我们认出它其实是惯性项那个抛物面碗的圆心——是被优化目标记在案的「锚点」，不只是一个起跑线。

代码里的 y：forwardStep() 那两行

这不是黑板上的符号，TinyVBD 的 forwardStep() 逐字算了它。先给速度加上重力冲量、把 pin 住的 0 号顶点速度清零，然后：

main.cpp
strand.mVelocity.colwise() += params.dt * params.gravity;
strand.mVelocity.col(0).setZero();
strand.inertia = strand.mVertPos + strand.mVelocity * params.dt;

注意第三行 inertia = mVertPos + mVelocity*dt：因为上一行刚把重力 $h\mathbf a_{ext}$ 加进了 mVelocity，这里乘以 dt 一并展开，正好就是 $\mathbf x^t+h\mathbf v^t+h^2\mathbf a_{ext}=\mathbf y$。代码把它存成成员 strand.inertia，后面 4.3 的 solve() 里那句 f = m*(inertia - x)*dtSqrReciprocal 算的就是惯性项的负梯度。所以 strand.inertia 这个变量，名副其实地装着我们的 $\mathbf y$。

系数 m/h²：谁把碗压得更陡

把惯性项对位置求两次导，它的 Hessian 是 $\frac{m}{h^2}\mathbf I$——一个各向同性、恒正定的碗，陡峭程度由标量 $\frac{m}{h^2}$ 决定。这个系数是整场拔河的胜负手：

  • 质量 $m$ 大、或时间步 $h$ 小 → 系数 $\frac{m}{h^2}$ 大 → 惯性碗又深又陡 → $\arg\min$ 被牢牢钉在 $\mathbf y$ 附近，弹性能再怎么喊「回 rest 形状」也拽不太动。直觉上对：重的东西惯性大、难被弹簧改向；小步长里物体来不及偏离自然轨迹。
  • 刚度 $k$ 大 → 弹性碗陡 → $\arg\min$ 被拽回「所有弹簧等于 rest length」的位形，哪怕这意味着大幅偏离惯性预测的 $\mathbf y$。硬弹簧说一不二。

所以「谁赢」从来不是绝对的，而是 $\frac{m}{h^2}$ 与 $k$ 的较量。同一根弹簧，挂个轻球它几乎拉不动惯性（球径直漂向 $\mathbf y$），挂个重球它却能把球硬生生拽回 rest 长度——因为重球那项 $\frac{m}{h^2}$ 把惯性碗压得太深。这就是为什么一根头发上「轻的中段」和「重的末端」在同一步里表现得判若两物。

这也解释了隐式欧拉的「数值阻尼」从哪来

1.1 说隐式欧拉会偷偷吃能量。从拔河看就很自然：每一步，最优解都是惯性预测 $\mathbf y$ 和弹性 rest 之间的折中，谁也没完全得逞。物体既没沿惯性走满、也没完全弹回——这份「各退一步」累积起来，宏观上就表现为振幅被一点点磨小，即数值阻尼。$h$ 越大，惯性碗越浅（$\frac{m}{h^2}$ 越小），折中越偏向弹性，阻尼越明显。

当质量悬殊：拔河变成一边倒

现在把两个质量差异极大的顶点放进同一根 strand——比如 TinyVBD 默认的 initializeTilted()，前 19 个顶点 $m_0=1$、末端 $m_1=1000$（high mass ratio $1000{:}1$）。轻顶点的惯性碗浅（$\frac{m}{h^2}$ 小），弹簧轻松摆布它；重末端的惯性碗深得离谱，连 $10^8$ 刚度的弹簧都未必一步拽得动它。在同一次求解里，这两类顶点的「拔河力量对比」差了三个数量级。

能不能在这种悬殊下还稳稳收敛，是区分各路解法的试金石。剧透一点 5.1：XPBD 那类按逆质量 $1/m$ 分配修正的对偶解法，会被重物压垮（轻物分到的修正过大而乱跳）；而 VBD 是直接解每个顶点位置的primal 方法，每个顶点用自己真实的 $\frac{m}{h^2}$ 当锚，质量越悬殊反而各算各的、互不拖累。这场「惯性 vs 弹性」的拔河，正是高质量比实验的微观舞台。

从 PBD 看 PBD 把惯性碗拆了，所以它没有「真正的锚」

回忆 1.2 的那笔账：PBD 只顾把约束推到零，相当于只留弹性碗、丢掉惯性碗。没有 $\frac{m}{h^2}$ 这个锚，拔河就成了「弹性一方自说自话」——迭代越多越往纯弹性最优滑，于是「有效刚度随迭代次数变化」。换句话说，PBD 之所以病，正是因为它单方面退出了这场拔河。VBD 把惯性碗原封不动留着，拔河才是真拔河，解才钉得住、收敛到一个良定义的位形。

拔河也告诉你：好初值该放哪

既然每步都要把 $G$ 这个碗推到谷底，从哪儿起步就很关键——起点离谷底越近，迭代越省。一个自然的猜测是直接从惯性预测 $\mathbf y$ 出发（自由下落的顶点理应大致漂到那）。但对一个被弹力或接触托住的顶点，$\mathbf y$ 反而是个坏起点：它假设顶点会自由下落，可实际上它根本没动，从 $\mathbf y$ 起步等于先把它扔下去再费劲拉回来。

TinyVBD 的 forwardStep() 因此做了更聪明的adaptive initialization：用上一帧的近似加速度在重力方向的投影，逐顶点决定初值要带上多少重力——自由下落的顶点几乎带满（从 $\mathbf y$ 起步），被托住的顶点几乎不带（停在原位起步）。这套自适应初值的细节留到 4.2，但它的动机此刻就清楚了：它在为这场拔河挑一个尽量靠近谷底的发令枪位置。顺带一提，VBD 能这么自由地挑初值，本身就是个优势——它不像 XPBD 那样被求解结构强制要求某种初始化。

这一节你要带走的

三句话：(1) 预测位置 $\mathbf y=\mathbf x^t+h\mathbf v^t+h^2\mathbf a_{ext}$ 是惯性碗的圆心，就是 forwardStep() 里 strand.inertia 那一行；(2) 惯性系数 $\frac{m}{h^2}$ 与刚度 $k$ 较量碗的陡峭——重质量/小步长偏向惯性、大刚度偏向 rest 形状，胜负随质量比剧烈改变；(3) 这场拔河既预告了高质量比实验（5.1），也解释了为什么需要一个好初值（4.2）。

下一节 1.4，我们把抽象的弹性能 $E(\mathbf x)$ 落到一根真实的 Hookean 弹簧上，亲手算出它的能量、力和 Hessian——并撞见一个关键现象：弹簧被压缩时，Hessian 会变得「不定」。这正是 VBD 局部求解必须直面的硬骨头。
