TinyVBD ↔ Newton ↔ Isaac 对照速查

这一页把三套世界并排放在一起：你已经吃透的那根 TinyVBD strand、生产级的 Newton particle VBD、以及你从 EBench 熟悉的 Isaac / OpenUSD。每一行都在回答同一个问题——什么没变，什么变了。贯穿全表的那条没变的线，就是变分目标 $\min_{\mathbf x} G(\mathbf x)$；变的只是「一根 strand 串行 → 一块 cloth 在 GPU 上按颜色并行」。把它当成翻一眼就能定位概念的对照表用。

怎么读这张表

第二列「TinyVBD」是你的锚点（原书 Part 1–6 的那 400 行 2D strand）；第三列「Newton particle VBD」是 GPU 上的 3D 推广；第四列是把概念接回 Isaac/USD 的旁注，— 表示没有干净的对应物（多为 VBD / 约束求解特有的东西）。术语解释见 Newton 术语表 与 原书术语表，公式见 公式速查。

仿真对象与数据组织

先对齐「我们在仿真什么、数据放在哪」。最大的结构差异：TinyVBD 把一切塞进一个对象，而 Newton 把静态（Model）和动态（State）拆开——这正好对上 Isaac 里「静态资产 vs 动态属性」的分法。

从 TinyVBD 看 一个对象 vs Model+State

原书里位置和速度都是 VBDStrand 的字段，读写随手就来，因为只有一根 strand、只在 CPU 上跑。Newton 要在 GPU 上同时跑很多场景、还要可微，所以把「永远不变的」（拓扑、质量、颜色）冻进 Model，把「每个时刻在变的」（$\mathbf x,\mathbf v,\mathbf f$）放进可交换的 State 快照。换句话说：你的 mVertPos 被拆成了「Model 里的身份」+「State 里的当前值」。

算法核心：初始化 · 求解 · 加速

这是全书主线最该并排看的地方。目标函数 $G(\mathbf x)=\frac{1}{2h^2}\lVert\mathbf x-\mathbf y\rVert_M^2+E(\mathbf x)$ 在三列里完全没变；每个顶点的牛顿步 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$ 也没变（只是从 2×2/3×3 解法换了具体数值实现）。诚实地说，有两处 Newton particle 路径反而比 TinyVBD 简单：它没有自适应初始化、也没有 Chebyshev。

从 TinyVBD 看 两处「Newton 反而更朴素」

你可能以为生产引擎处处更花哨——但 particle VBD 在这两点上比原书简单：(1) 初始化只用朴素惯性 $\mathbf y=\mathbf x+(\mathbf v+\mathbf g\,h)\,h$，没搬你那套 accelerationComponent 自适应；(2) 它不用 Chebyshev。原因是 Newton 靠 GPU 上海量并行 + 多 color sweep 把收敛压下去，工程上选择不引入 Chebyshev 那点谱半径 $\rho$ 的调参负担。$\min G$ 没变，省掉的只是两个「加速/预热」技巧。

物理元件 · 碰撞 · 并行

真正「长大」的地方在物理建模和并行：strand 只有弹簧、没有碰撞、串行扫一遍；cloth/soft 有一整套 Neo-Hookean 元件 + 自碰撞，并且按 graph coloring 在 GPU 上并行。注意：弹簧的力/Hessian 形式两边一模一样——这是把概念接回 strand 最自然的接口。

从 TinyVBD 看 spring 是公共子集

别被 Neo-Hookean 吓到：Newton 的 spring 元件用 spring_rest_length / stiffness / damping，力和 Hessian 的代数形式和你原书弹簧逐项相同，只是从 2D 升到 3D。membrane / volumetric / bending 是在这个公共子集之外、为面/体单元新加的本构；碰撞和着色则是你那根孤零零的 strand 从来没遇到过的新问题。

IPC 不在这张表里——它是概念篇

上面「碰撞」行里的 Newton 列是 BVH self-collision + penetration-free 截断；Isaac 列是 PhysX。IPC 不是 Newton/Isaac 代码里的求解器——Newton 的 particle/rigid 用 VBD/AVBD（10-3 的 local barrier + DAT、10-x 的 penalty/AVBD）。Part 12 是算法/概念篇，单独讲第三条接触路线（interior-point barrier + CCD 严格 intersection-free），所以它没有「Newton 代码列」可填。出处见 a-7 IPC 参考文献。

接触的三条路线（penalty / AVBD / IPC）

把「碰撞」这一格再拆细：本书一路走过三种接触哲学，软硬与保证强度递增（对照 12.11）。

一句话主线：什么没变 / 什么变了

把整张表压成一句：

$$\underbrace{\min_{\mathbf x} G(\mathbf x)}_{\text{没变}}\quad\text{；}\quad\underbrace{\text{一根 strand 串行}\ \longrightarrow\ \text{一块 cloth 在 GPU 上按颜色并行}}_{\text{变了}}$$

没变：变分目标 $\min G$、每顶点局部牛顿 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$、惯性项 $\frac{m_i}{h^2}\mathbf I$ 的正定结构、弹簧的力/Hessian 形式、后向差分更新速度 $\mathbf v=(\mathbf x-\mathbf x_{prev})/h$。

变了：单对象 → Model/State 拆分；2D → 3D；仅弹簧 → membrane/volumetric Neo-Hookean + bending + spring；无碰撞 → BVH 自碰 + penetration-free 截断；串行单 sweep → graph coloring 多色并行；自适应初始化 → 朴素惯性；有 Chebyshev → 无 Chebyshev。

读到这里，你应该能拿着任意一个 Newton particle VBD 的概念，反手指回原书那根 strand 上的对应物——这正是这一页存在的意义。完整术语见 Newton 术语表，外部出处见 Newton 参考；接触的第三条路线 IPC 是概念篇，出处见 a-7 IPC 参考。
