Newton / Warp 术语表

Newton 篇里冒出来一大批 Warp 与 Newton 的新名字——kernel、coloring、Neo-Hookean、augmented Lagrangian……这张表把 Part 7–10 引入的术语集中起来，每个配一句中文解释和「第一次认真讲到」的链接，方便你边读代码、边查论文时随时回查。隔壁 术语表 Glossary 收的是 TinyVBD 原书的术语，两张表互补。

Warp 编程模型

术语中文解释详见
Newton 建在 NVIDIA Warp 之上的 GPU 物理引擎，面向 robotics 与仿真研究；由 Disney Research / Google DeepMind / NVIDIA 发起，现为 Linux Foundation 项目，Apache-2.0。 7.1
NVIDIA Warp Newton 底层的 Python 框架：用 Python 写、即时编译成 GPU kernel 的 SPMD 计算库；Newton 扩展并泛化了它已弃用的 warp.sim。 7.2
SPMD Single Program, Multiple Data——同一段 kernel 程序在成千上万个 thread 上并行跑，每个 thread 只处理自己那一份数据。Warp 的执行模型。 7.2
@wp.kernel 并行入口装饰器；被 @wp.kernel 标记的函数会被编译并由 wp.launch 在 N 个 thread 上同时启动。 7.2
@wp.func device 函数装饰器（类似 CUDA __device__），可被 kernel 调用、不能单独 launch；力 / Hessian 装配函数都是 @wp.func。 8.4
wp.array GPU 上的数组，带 dtype（如 wp.vec3 / wp.float32），是 Model / State 里一切物理量的存储载体。 7.2
wp.launch 启动 kernel 的调用：wp.launch(kernel, dim=N, inputs=[...]) 拉起 N 个 thread，dim 就是并行度。 7.2
wp.tid() kernel 内取当前 thread 索引；常见第一行 i = wp.tid()，用它去 index 自己负责的那个粒子 / 元件。 7.2
wp.vec3 / wp.mat33 Warp 内置的 3 维向量 / 3×3 矩阵类型；per-vertex 牛顿步里的 $\mathbf f_i$ 是 wp.vec3、$\mathbf H_i$ 是 wp.mat33。 8.4
wp.struct 把若干数组 / 标量打包成一个结构体的装饰器，方便整块传进 kernel；如邻接信息 ParticleForceElementAdjacencyInfo。 8.4
CUDA graph capture 把整段仿真循环录成一张 CUDA graph 再零开销重放：with wp.ScopedCapture() as capture: ... 配 wp.capture_launch(capture.graph)，省去每帧重复的 launch 开销。 7.4
MuJoCo Warp MuJoCo 在 Warp 上的实现，被 Newton 集成为主要后端之一（用于 rigid / 关节动力学）。 7.1
OpenUSD Pixar 的通用场景描述格式；Newton 原生支持 USD，与读者熟悉的 Isaac/USD 资产管线一脉相承。 7.1

Newton 数据模型

术语中文解释详见
Model 静态场景：粒子 / 刚体 / 关节 / 形状及其物理属性，数组按 id 索引（particle_q、particle_mass、gravity 等）。一次 finalize 后基本冻结。 7.3
State 某一时刻的动态快照：位置 / 速度 / 力。VBD 在 state_in 与 state_out 之间来回 ping-pong。 7.3
Control 控制输入容器：joint_f、joint_target_q 等驱动量，按帧喂给 solver。 7.3
Contacts 碰撞检测的输出容器，由 model.collide(state, contacts) 填充，再交给 solver 算接触力。 7.3
ModelBuilder 搭场景的构造器：往里加粒子 / cloth / 形状，最后 finalize() 冻成 Model。 7.4
finalize() builder.finalize()：把 builder 里堆好的场景冻结成不可变的 Model，之后才能造 solver / state。 7.4
builder.color() 给 particles 与 rigid bodies 做 graph coloring，结果存进 particle_color_groups / body_color_groups；VBD 必需这一步。 8.5
particle_color_groups / body_color_groups 着色结果：按颜色分组的顶点 / 刚体索引表；同组之间不共享 force element，可并行更新。SolverVBD 构造时检查 particle 版存在。 8.5
particle_q / particle_qd State 里的粒子位置数组（wp.array[wp.vec3]）与速度数组；对应 TinyVBD 里的 $\mathbf x$ 与 $\mathbf v$。 7.3
solver.step() 求解器契约 solver.step(state_in, state_out, control, contacts, dt)：推进一个时间步，内部走 Initialize → Iterate → Finalize 三段式。 8.2

VBD 弹性与碰撞

术语中文解释详见
graph coloring (primal / dual) 给图染色使相邻元素不同色。VBD 着的是顶点图（primal），颜色远少于 XPBD 那种约束图（dual）；同色可并行。 8.5
Gauss-Seidel by color 按颜色一组一组地扫：组内并行、组间串行，立刻用上前一组的新值。Newton 的 particle 路径就是纯粹这一套，没有 Chebyshev 加速。 8.5
Neo-Hookean (membrane / volumetric) 超弹性能量模型。membrane 版给 cloth 三角（变形梯度 $\mathbf F$ 是 3×2）；volumetric 版给 tet soft body（$\mathbf F$ 是 3×3）。 9.1
stable Neo-Hookean ($\lambda_{NH}=\lambda+\mu$) Smith 2018 的稳定版 Neo-Hookean 约定：用 $\lambda_{NH}=\lambda_{\text{Lamé}}+\mu_{\text{Lamé}}$、能量 $\psi=\frac{\mu}{2}(I_c-2)+\frac{\lambda}{2}(J_s-\alpha)^2$，$\alpha=1+\mu/\lambda$。 9.1
dihedral bending 二面角弯曲能量：相邻两三角的夹角 $\theta$ 偏离静止角的代价 $E=\frac12 (k\,L_{\text{rest}})(\theta-\theta_{\text{rest}})^2$；边界边跳过。cloth 的弯曲刚度来源。 9.1
SPD projection 把局部 Hessian 投影成对称正定，保证每步牛顿都是「往谷底走」。membrane 的做法是把违正定的特征值 clamp（s_clamp = max(0, …)）。 9.2
Rayleigh damping ($D=k_d\cdot k_e$) 瑞利阻尼：阻尼系数取刚度的常数倍，$D=k_d\cdot k_e$（$k_d$ 是无量纲 Rayleigh 系数）；membrane / volumetric 各有 Frobenius / trace 形式的耗散项。 9.2
self-collision / BVH 自碰撞检测：用 BVH（bounding volume hierarchy）做 broad-phase，再做 vertex-triangle 与 edge-edge 的 narrow-phase；TinyVBD 完全没有碰撞。 9.3
penetration-free truncation (DAT) 每色 sweep 后，按保守界把这一步的位移截断，保证不穿透；是 production 稳定性的关键一环。 9.3

AVBD 与 rigid

术语中文解释详见
VBD (Vertex Block Descent) 本书主角，搬到 GPU 上：per-vertex 3×3 牛顿步 + 按颜色的 Gauss-Seidel，无条件稳定。Newton 的 particle 路径用它。 8.1
AVBD (Augmented VBD) VBD 的增广版（Giles 2025）：在 VBD 上加 augmented Lagrangian 与自适应 primal-dual，能强制硬约束、并在高刚度比下收敛更好。Newton 的 rigid 路径用它。 10.1
augmented Lagrangian 增广拉格朗日：在惩罚项之外再带一个拉格朗日乘子 $\lambda$，使约束能被精确满足而不必把惩罚刚度推到无穷。AVBD 处理硬约束的核心。 10.1
penalty ramping ($k_{start}$ / $\beta$ / $\gamma$) 惩罚刚度的自适应演化：每帧从 per-constraint 的 $k_{start}$ warm-start，迭代内对仍违反的约束以 $\beta$ 增长、跨帧以 $\gamma$ 衰减；soft 约束封顶在材料刚度，hard 约束可冲向更高。论文默认 $\alpha=0.95,\beta=10,\gamma=0.99$。 10.1
