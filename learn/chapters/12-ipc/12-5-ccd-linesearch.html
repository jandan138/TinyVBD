intersection-free 的真正来源：CCD-filtered line search

前几节我们把 IPC 的 log-barrier 接触能量、它的距离/梯度/Hessian 都摆好了——能量在 $d\to0$ 时发散到无穷，理论上「距离永远跨不过零」。但理论是连续的，求解是离散的：一步 Newton 迈得太大，再陡的能量墙也可能被一脚跨过去。这一节讲 IPC 真正把 intersection-free 从「应该不会穿」升级成「保证不穿」的那台机器——CCD-filtered line search。它也是整个 IPC 篇的心脏。

能量墙再陡，离散的一步也能跨过去

先把上一节的结论接过来：barrier $b(d,\hat d)$ 在距离 $d$ 趋近零时能量发散，对应的接触力可以变得任意大。在连续的意义上，这道墙确实拦得住任何东西——没有有限的力能把距离压到零。可一旦真正去解，你面对的是一个迭代：每一步先求出一个下降方向，再沿这个方向走一段。问题就出在「走一段」这三个字上。

设当前位形是 $\mathbf x$，全局 Newton 解出了一个方向 $\mathbf p$（怎么解出来的下一节再说，这里只当它是一个「往能量更低处去」的方向）。接下来要决定步长 $\alpha$，更新成 $\mathbf x + \alpha\,\mathbf p$。如果 $\alpha$ 取得太大，哪怕能量墙在 $d=0$ 处是无穷高，离散的这一跳也可能直接落到墙的另一侧——两个几何元件已经交叉，距离变成了「负」的，barrier 公式甚至失去意义。能量发散保证的是「连续轨迹跨不过」，它管不住「离散采样跳过去」。

这正是 10-3 你已经撞过的那堵墙

这个「一步跨过去」的风险，你在 10.3 已经亲眼见过。那里 Newton cloth 的 log-barrier 同样在能量层面挡穿透，但 VBD 是按 color 一遍遍扫的离散迭代，某一步算出的 $\Delta\mathbf x_i$ 可能步子太大一脚跨过三角形——所以 10-3 在 每个 color sweep 之后加了一道 penetration-free 截断（DAT）做几何兜底。同样的离散跨步风险，IPC 和 10-3 都有；区别全在「怎么解决它」。这一节就是讲 IPC 的解法。

第一道闸：filtered CCD 求最大安全步长

IPC 的办法不是「先走完再检查、穿了再砍」，而是在走之前就先问清楚：沿方向 $\mathbf p$ 最多能走多远而不发生任何交叉？回答这个问题的，是 CCD——continuous collision detection（连续碰撞检测）。

CCD 和我们熟悉的「离散碰撞检测」不同。离散检测只看某个时刻的快照：这一帧有没有交叉。CCD 看的是整段运动：把每个顶点从 $\mathbf x$ 到 $\mathbf x + \mathbf p$ 想象成一条直线轨迹，问「在这段连续扫掠的过程中，第一次发生 point-triangle 或 edge-edge 接触（距离归零）是在哪个比例 $t^\star\in(0,1]$」。这个 $t^\star$ 就是最大不穿透步长（time of impact）：只要 $\alpha \le t^\star$，整段运动里就没有任何一对几何元件穿过对方。

$$t^\star \;=\; \max\;\bigl\{\,t\in(0,1] \;:\; \mathbf x + s\,\mathbf p \ \text{对所有 } s\in[0,t]\ \text{都无交叉}\,\bigr\}$$

所谓 filtered，是说 CCD 不会傻乎乎对每一对几何元件都做这套连续求根——那是平方级灾难。它先用空间加速结构和保守的包围盒「过滤」掉绝大多数不可能在这步内相撞的元件对，只对真正逼近的少数几对做精确的 time-of-impact 计算。这一步是 IPC 每次迭代里实打实的开销，也是后面 12-6 要讲「IPC 贵在哪」的第一笔账。

interactiveCCD 求最大安全步长：点永远停在墙前
一个点朝线段运动，Newton 提出一个方向，CCD 求出 time-of-impact、line search 把步长截到可行域内——拖动方向/步长，点永远停在前面，绝不穿过。

第二道闸：CFL 式保守接触界

光有 $t^\star$ 还不够「安全」。如果每步都正好走到 $t^\star$，那就意味着这一步走完后某对元件的距离恰好压到了零——而 barrier 在 $d=0$ 处是发散的，下一步的能量、梯度、Hessian 会瞬间爆炸，数值上完全没法继续。IPC 因此在 $t^\star$ 之外再叠一道更保守的限制：一个 CFL 式的保守接触界，进一步把步长往回收一点，确保走完之后元件之间仍留着一条正的安全缝，距离停在 $\hat d$ 这道激活阈值能温柔接管的范围里，而不是贴死在零上。

「CFL 式」是个借喻：经典的 CFL condition（Courant–Friedrichs–Lewy）在显式时间积分里限制「一步走过的距离不能超过一个网格格子」，本质是「别让一步跨得比你能分辨的尺度还大」。IPC 这里同理——别让一步走得比「当前最近的那对元件之间还剩多少缝」更激进。两道闸合起来，最终步长是

$$\alpha \;=\; \min\bigl(\,\alpha_{\text{Newton}},\ \ c\cdot t^\star,\ \ \alpha_{\text{CFL}}\,\bigr),\qquad 0<c<1$$

其中 $\alpha_{\text{Newton}}$ 是普通 line search（保证能量真的下降、Armijo 之类）想要的步长，$c\cdot t^\star$ 是 CCD 给的不穿透上限（留了系数 $c$ 不贴死），$\alpha_{\text{CFL}}$ 是保守接触界。三者取最小——谁最保守听谁的。

为什么不直接走到 t* 再说

因为 barrier 不是「碰到才发力」，而是「在 $\hat d$ 之内就发力、越近越猛」。如果允许步长走到距离恰好为零，barrier 能量和它的导数都已发散，整个二次模型在那一点失效，Newton 没法在下一轮继续。保守界的作用是始终把系统留在 barrier 还能正常工作的「内部」——这正是 interior-point（内点法）这个名字的字面意思：永远待在可行域内部，绝不踩到边界上。

关键：可行性贯穿每一次非线性迭代

现在把它和 10-3 的差别钉死。10-3 的 penetration-free 截断是一步牛顿之后的几何动作：先按能量算出 $\Delta\mathbf x_i$，再回头按保守距离界把这批位移砍短——它是事后的、per-color 的、不可微的几何兜底。它管用，但它发生在求解步的外面，是给离散迭代加的一道补丁。

IPC 把这件事焊进了求解的内部。每一次 Newton 迭代提出方向后，line search 本身就带着 CCD 和保守界去挑步长——于是这一步走完，位形必然仍在可行域内（无交叉）。而一个 time step 往往要做很多次这样的非线性迭代才收敛；既然每一次迭代都从可行点出发、又落在可行点上，那么不管中间迭代了多少轮、压缩得多厉害，整条迭代链上的每一个位形都是 intersection-free 的。

从 TinyVBD 看 还是 min G(x)，只是 line search 学会了「看路」

别忘了这一切仍在主线 min G(x) 之内。IPC 没有另起炉灶——它最小化的还是一个 incremental potential，接触项还是那个 log-barrier。变的只是怎么沿方向走一步：普通优化里 line search 只关心「能量有没有下降」，IPC 的 line search 额外背上了一个硬约束——「这一步不许跨过任何几何边界」。CCD 求 $t^\star$、保守界再收一点，本质上是给 line search 装了一双能看见前方墙在哪的眼睛。主线没改，只是走路更小心了。

和 10-3、11-1 三方对照

把三条路在「无穿透」这一维上摆齐，差别一目了然：

  • 10-3 Newton cloth：log-barrier 软挡 + 每个 color sweep 之后的 penetration-free 截断（DAT）做事后几何兜底。无穿透靠的是「截断这道补丁」，发生在 per-vertex 局部求解步之外。
  • 11-1 AVBD：把非穿透当一道硬约束，靠 penalty stiffness 一路 ramp 向无穷去逼近它（见 11.1）。逼近得很近，但「保证」二字仍是渐近意义上的。
  • 11 IPC：把无穿透烤进全局求解的 line search。CCD-filtered line search + CFL 保守界让每一次迭代都停在可行域内部——于是 intersection-free 不是兜底、不是逼近，而是每一步都严格成立的不变量，与压缩程度无关。

同一个数学对象（log-barrier），同一个目标（不穿透），三种「怎么保证它」的工程哲学。IPC 最贵，但它给的是三者里唯一带严格保证的那一个。

这一节你要带走的

三句话：(1) barrier 在连续意义上挡穿透，但离散的一步可能跨过能量墙——10-3 和 IPC 都有这个风险，区别全在解法；(2) IPC 的解法是 CCD-filtered line search：用连续碰撞检测求出最大不穿透步长 $t^\star$，再叠一道 CFL 式保守接触界把步长进一步收紧，最终 $\alpha=\min(\alpha_{\text{Newton}},\,c\,t^\star,\,\alpha_{\text{CFL}})$，永远停在可行域内部；(3) 因为这道保证烤进了每一次非线性迭代（而非像 10-3 那样在求解步之外做事后截断），所以整个 time step 严格 intersection-free，与压缩程度无关——这就是 IPC「保证」二字的真正来源。

到这里 IPC 的「不穿透」这条主线讲完了。但真实接触从不只有「推开」，还有沿着接触面相互摩擦。理想 Coulomb 摩擦有一处 Newton 最讨厌的不连续阶跃——下一节 12-6：摩擦——lagged、光滑化的 Coulomb，我们看 IPC 怎么把摩擦也塞进同一个优化框架，并把那个阶跃磨成可微的曲线。
