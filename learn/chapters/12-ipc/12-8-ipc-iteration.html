IPC 一次完整迭代：伪代码逐段走读

前面把每个零件单独打磨好了——现在把它们装进一台机器，看一次 IPC 迭代从头到尾怎么转。我们仿照 4-3 走读 solve() 的方式，把这台机器的伪代码一段段拆开。

先看全貌

4-3 里我们把 VBD 的 solve() 整段放出来再逐块拆。这里也一样，先把 IPC 一次外层 Newton 迭代的伪代码完整摆出来，然后一行一行对回前面几节的公式与章节。注意：

读法说明：这是伪代码，不是真实 IPC 源码

下面这段是为讲清结构而写的伪代码（Python 风格），不是 IPC 的真实实现。真实 IPC 里每一行都对应成百上千行高度优化的 C++（稀疏装配、并行 CCD、CHOLMOD 调用…）。我们要的是「手指能点到每一步」的骨架，不是可编译的代码。变量名也只为对应公式，不代表论文里的命名。

pseudocode · 一次 IPC 外迭代
# 伪代码，非真实 IPC 代码，仅示意结构
# 输入：当前位形 x（已在可行域内，无交叉）；惯性目标 y
# 输出：更新后的 x，仍无交叉；是否收敛

def ipc_outer_iteration(x, y):
    # 1) 装配全局稀疏 H, f：惯性 + 弹性 + barrier 三部分贡献
    #    梯度 ∇G → f（取负），Hessian ∇²G → H
    H, f = assemble_global(x, y)          # 累加 inertia / elasticity / barrier

    # 2) 对每个局部 Hessian 块做 PSD 投影，再装进全局 H
    H = psd_project_blocks(H)             # 负特征值钳到 0，保证下降方向

    # 3) 解全局稀疏线性系统 H p = f，得到下降方向 p
    p = sparse_cholesky_solve(H, f)       # CHOLMOD：符号分解 + 并行数值分解

    # 4) filtered CCD：沿 p 求最大不穿透步长 t*（time of impact）
    t_star = filtered_CCD(x, p)           # 整段扫掠里第一次接触的比例

    # 5) line search：在 [0, 上限] 内挑既下降又不越界的步长
    alpha = line_search(x, p, t_star)     # = min(alpha_newton, c*t_star, alpha_cfl)

    # 6) 接受这一步；判收敛
    x = x + alpha * p
    converged = norm(grad_G(x)) < tol     # 梯度足够小 → 已到极小附近
    return x, converged

骨架是：装配全局 H/f → PSD 投影 → 全局求解出方向 p → CCD 求安全步长 → line search 定最终步长 → 接受并判收敛。把它和 4-3 的 VBD solve() 并排看，会发现两者目标完全相同（都在最小化同一个 incremental potential $G$），但「怎么走一步」的机器截然不同。下面逐块拆。

从 TinyVBD 看 同一个 min G(x)，两台不同的机器

4-3 的 VBD solve() 是一个外层循环扫每个顶点，每个顶点解一个 3×3 牛顿步、就地写回——这是 Gauss-Seidel 式的 per-vertex 局部下降。IPC 这段伪代码没有「扫顶点」的外层循环：它一次性装配出整个系统的稀疏 $\mathbf H$、$\mathbf f$，做一次全局求解得到所有顶点的方向 $\mathbf p$。同样是把 $G$ 往下推，VBD 是「一个一个顶点轮流推」，IPC 是「所有顶点一起、解一个大方程推」。同一个 $\min G(\mathbf x)$，不同的机器。

第 1 段：装配全局 H、f

① assemble_global
H, f = assemble_global(x, y)   # 累加 inertia / elasticity / barrier

这一行对应 4-3 里给每个顶点填 f、h 的那两段——只是 IPC 把所有顶点、所有项一次性装进全局稀疏矩阵。$G$ 的梯度取负得到 f，$G$ 的 Hessian 得到 H：

$$\mathbf f = -\nabla G(\mathbf x),\qquad \mathbf H = \nabla^2 G(\mathbf x)$$

$G$ 由三部分相加，装配时也就累加三份贡献：

  • 惯性项：和 4-3 同源的 $\frac{m_i}{h^2}(\mathbf y_i-\mathbf x_i)$ 进 f、$\frac{m_i}{h^2}\mathbf I$ 进 H（对角块，恒正定）。这是那块「定海神针」。
  • 弹性项：每个 force element（弹簧 / 三角形 / 四面体）按它的能量贡献力与 Hessian，落到相关顶点的块上——就是 4-3 内层循环累加 h_1_1 和带 ± 号的力那件事，只不过现在写进全局矩阵的对应位置。
  • barrier 项：这是 IPC 独有、也是这一段最关键的部分。遍历当前所有逼近的接触对（point-triangle / edge-edge），对每一对算出 barrier 的力与 Hessian。

barrier 那一份正是 12-4 推的链式法则：barrier 是距离 $d$ 的函数，距离又是顶点位置的函数，所以

$$\frac{\partial b}{\partial \mathbf x} = b'(d)\,\frac{\partial d}{\partial \mathbf x},\qquad \frac{\partial^2 b}{\partial \mathbf x^2} = b''(d)\,\frac{\partial d}{\partial \mathbf x}\frac{\partial d}{\partial \mathbf x}^{\!\top} + b'(d)\,\frac{\partial^2 d}{\partial \mathbf x^2}$$

其中 $b'$、$b''$ 来自 12-2 的 barrier 解析式，距离梯度 $\partial d/\partial \mathbf x$ 来自 12-3——还记得吗？它按 barycentric 权重落到那对元件涉及的几个顶点上（point 端拿 $\hat n$，三角形三个顶点各拿 $-\lambda\hat n$）。装配就是把这每一对接触的小贡献，加到全局 $\mathbf H$、$\mathbf f$ 里属于这几个顶点的行列上。

谁参与装配，由「逼近」决定

不是所有元件对都进 barrier 装配——只有距离 $d < \hat d$（落进激活阈值）的接触对才有非零贡献，正是 12-2 那条「$d\ge\hat d$ 时 $b=0$」。所以每轮迭代前要先用空间加速结构找出这批逼近的对（和 12-5 里 filtered CCD 的「过滤」是同一套加速思想）。这份「当前活跃接触集」会随着 $\mathbf x$ 变化而变。

第 2 段：PSD 投影每个局部块

② psd_project_blocks
H = psd_project_blocks(H)   # 负特征值钳到 0，保证下降方向

这一段在 4-3 的 VBD solve() 里是没有的——VBD 故意不投影，靠那块恒正定的惯性 Hessian 把整体兜回正定附近（见 3.6）。IPC 走全局 Newton，就必须更讲究：要让 $\mathbf H\,\mathbf p=\mathbf f$ 解出来的 $\mathbf p$ 确实是一个下降方向，$\mathbf H$ 得是（半）正定的。

问题出在哪？弹性 Hessian 在压缩时可能不定（4-3 里那个 $k(1-l_0/l)$ 负特征值），barrier Hessian 里那个 $b'(d)\,\partial^2 d/\partial\mathbf x^2$ 项（距离的二阶项）也不保证半正定——这正是 12-4 重点讲的。办法和 10-2 的 SPD 投影同源：对每个局部 Hessian 块做特征分解，把负特征值钳到零，再把投影后的块装进全局 $\mathbf H$。

$$\mathbf H_{\text{local}} = \mathbf Q\,\mathrm{diag}(\lambda_1,\dots,\lambda_n)\,\mathbf Q^\top \;\longrightarrow\; \mathbf Q\,\mathrm{diag}(\max(\lambda_1,0),\dots,\max(\lambda_n,0))\,\mathbf Q^\top$$

「逐块投影再装配」是关键细节：投影发生在每个小块（一个 force element、一对接触）本地，而不是对整个全局大矩阵做（那太贵）。每个本地块半正定，累加成的全局 $\mathbf H$ 也半正定——于是下一步的解必为下降方向。

第 3 段：全局稀疏求解出方向 p

③ sparse_cholesky_solve
p = sparse_cholesky_solve(H, f)   # CHOLMOD：符号分解 + 并行数值分解

这是 IPC 和 VBD 分道扬镳最明显的一行。4-3 里求方向是 dx = h.colPivHouseholderQr().solve(f)——一个 3×3 的小系统，每个顶点一个，互相独立。IPC 这里是一个覆盖全部自由度的大稀疏系统：

$$\mathbf H\,\mathbf p = \mathbf f$$

$\mathbf H$ 是 $3N\times 3N$（$N$ 个顶点）的稀疏矩阵，解它要用稀疏直接法。IPC 用 CHOLMOD 做 Cholesky 分解：先按 $\mathbf H$ 的稀疏结构做一次符号分解（算出非零位置、可在结构不变时复用），再做并行数值分解得到 $\mathbf p$。因为第 2 段已把 $\mathbf H$ 投成半正定，Cholesky 才用得上（它要求矩阵正定）。

这一步贵在哪，下一节专门算账

「解一个全局稀疏系统」正是 IPC 比 VBD 的 per-vertex 3×3 贵的核心来源之一——它不能像 VBD 那样天然 per-vertex 并行，而要做一次有数据依赖的稀疏分解。这笔账连同 CCD 的开销，12-9 会把 IPC「稳但贵」的全景摊开。这里先记住：解出的 $\mathbf p$ 是「往 $G$ 更低处去」的方向，但还不是最终位移——步长由下面两段决定。

第 4、5 段：CCD 求安全步长 + line search 定步长

④⑤ filtered_CCD + line_search
t_star = filtered_CCD(x, p)        # 整段扫掠里第一次接触的比例
alpha  = line_search(x, p, t_star) # = min(alpha_newton, c*t_star, alpha_cfl)

有了方向 $\mathbf p$，要决定沿它走多远。这正是 12-5 整节讲的那台机器，这里把它接进迭代：

  • filtered_CCD(x, p)：沿方向 $\mathbf p$ 把每个顶点的运动想成一段直线扫掠，用 filtered CCD 求出最大不穿透步长 $t^\star\in(0,1]$——整段运动里第一次发生接触（距离归零）的比例。只要走的步长 $\le t^\star$，这一步就没有任何一对元件穿过对方。
  • line_search(x, p, t_star)：在不穿透的前提下，再挑一个真能让 $G$ 下降的步长。最终步长是三者取最小：
$$\alpha = \min\bigl(\,\alpha_{\text{Newton}},\ \ c\cdot t^\star,\ \ \alpha_{\text{CFL}}\,\bigr),\qquad 0

其中 $\alpha_{\text{Newton}}$ 是普通 line search（Armijo 之类，保证能量真的降）想要的步长；$c\cdot t^\star$ 是 CCD 给的不穿透上限，乘 $c$ 是为了不贴死在 $d=0$（barrier 在那里发散）；$\alpha_{\text{CFL}}$ 是 12-5 那道 CFL 式保守接触界，留一条正的安全缝。谁最保守听谁的。

第 6 段：接受这一步 + 收敛判据

⑥ accept & converge
x = x + alpha * p
converged = norm(grad_G(x)) < tol   # 梯度足够小 → 已到极小附近

对应 4-3 的 strand.vertex(iV) += dx——只是这里是整批顶点一起按同一个 $\alpha$ 沿 $\mathbf p$ 挪：

$$\mathbf x \;\leftarrow\; \mathbf x + \alpha\,\mathbf p$$

因为 $\alpha$ 被 CCD + 保守界压在可行域内，这一步走完 $\mathbf x$ 仍然无交叉——而且我们是从一个无交叉的 $\mathbf x$ 出发的。于是「无交叉」是一个贯穿每次迭代的不变量：进来是可行的，出去还是可行的。这正是 12-5 反复强调的、IPC「保证」二字的来源。

最后判收敛：当梯度 $\lVert\nabla G(\mathbf x)\rVert$ 已经足够小（小于阈值 tol），说明 $\mathbf x$ 落到了 $G$ 的极小附近，这个 time step 的非线性求解就结束；否则带着新的 $\mathbf x$ 回到第 1 段，重新装配、再来一轮。

「一次迭代」和「一个 time step」的关系

和 4-3 一样要分清层级：上面这段伪代码是一次外层迭代。一个 time step 通常要跑很多次这样的迭代，直到 converged 为真——就像 4-3 的 solve() 一次只走一遍 sweep，主循环要调它很多次。区别是：VBD 每次「一遍 sweep」很便宜、要跑很多遍；IPC 每次外迭代很贵（全局解 + CCD），但每次都把整个系统往极小推一大步。每一次外迭代结束都 intersection-free，所以中间无论迭代多少轮、压缩多厉害，整个 time step 始终无交叉。

把六段连起来看：单步走一遍流水线

把这六段连成一条流水线动起来，最容易看清「方向 $\mathbf p$ 提议的 full step 可能穿墙，但 $\alpha$ 把它截在墙前」这件事：

interactive一次 IPC 外迭代的流水线：assemble → PSD → solve → CCD → line-search → accept
点「单步」逐阶段走读这条流水线，看左侧高亮从装配一路推进到接受；同时盯右侧的点-墙小场景：solve 阶段提出的 full step（虚线）可能一脚跨过墙，但走到 CCD / line-search 阶段后，实际的 $\alpha$ 步（实线）永远停在墙前。这就是六段伪代码合起来在做的事。

这一节你要带走的

三句话：(1) IPC 一次外迭代的骨架是 装配全局稀疏 $\mathbf H,\mathbf f$（惯性+弹性+barrier）→ 逐块 PSD 投影 → 全局稀疏 Cholesky 解出方向 $\mathbf p$ → CCD 求 $t^\star$ → line search 定 $\alpha=\min(\alpha_{\text{Newton}},c\,t^\star,\alpha_{\text{CFL}})$ → $\mathbf x\!\leftarrow\!\mathbf x+\alpha\mathbf p$ 并判收敛，每一段都接得回 12-2/12-3/12-4/12-5；(2) 它和 4-3 的 VBD solve() 目标相同（都是 $\min G(\mathbf x)$）但机器不同——VBD 是 per-vertex Gauss-Seidel sweep，IPC 是一次全局求解；(3) 因为 $\alpha$ 被 CCD 与保守界焊在可行域内，每一次外迭代结束都无交叉，于是 intersection-free 是贯穿整个 time step 的不变量。
