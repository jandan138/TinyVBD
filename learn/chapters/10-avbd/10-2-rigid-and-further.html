rigid AVBD 一瞥与进一步阅读

上一节（10.1）我们看清了 augmented Lagrangian 怎么把硬约束和高刚度比塞进 VBD 的下降骨架。这一节是全书最后一站：先到 Newton 的 rigid 一侧瞄一眼——同一套 AVBD 思想是怎么去管刚体的 joint 和 contact 的；然后摊开一份进一步阅读清单，最后合上这本书。我们不深挖 rigid kernel，只看它和你已经吃透的那根 strand 是怎么一脉相承的。

SolverVBD 的另一半：rigid 用 AVBD

从第一节 Newton 篇起，我们一直只看 SolverVBD 的 particle 那一半——粒子、布料、软体，全是 per-vertex 的 VBD。但这个求解器的类 docstring 早就写明了它的双重身份：对 particle 用 VBD，对 rigid body 用 AVBD。换句话说，上一节那套 augmented Lagrangian 不是停在纸面上的理论，它就是 Newton 用来积分刚体的引擎。

rigid 这一侧要处理的东西，和粒子很不一样。粒子的状态是位置 $\mathbf x\in\mathbb R^3$；刚体的状态是一个 transform（平移 + 旋转，body_q: wp.array[wp.transform]）和一个 spatial_vector 速度（body_qd）。它们之间要满足的也不再是弹簧 rest length，而是两类典型约束：

  • joint（关节）：把两个刚体（或刚体与世界）按某种自由度铰接起来。Newton 的 AVBD 路径支持一组关节类型——BALL（球铰）、FIXED（固连）、FREE（自由）、REVOLUTE（转动）、PRISMATIC（移动）、D6（六自由度通用）、CABLE（缆绳）。每种关节把哪几个 DOF 锁住、哪几个放开，就决定了它要往能量里加哪些约束项。
  • contact（接触）：刚体之间、刚体与粒子之间的碰撞，从 collision pipeline（9.3）里来，作为接触约束进入同一个最小化问题。

无论 joint 还是 contact，AVBD 的处理方式都在上一节见过：每个约束挂一个 penalty 刚度 $k$ 和一个 Lagrange multiplier $\lambda$，按 hard / soft 两种 slot 来管理。soft slot 把约束当作软的惩罚项、封顶在某个刚度；hard slot 则通过 augmented Lagrangian 的对偶变量把约束往「精确满足」上逼。每帧 warm-start 时 penalty 按 $\gamma$ 衰减、迭代内按 $\beta$ 增长——和粒子侧的 penalty ramping 是同一套机制，只是作用在刚体的 transform 上。

从 TinyVBD 看 一根弹簧 → 一个 joint

别被「刚体」吓到。回想 TinyVBD 里那根 strand：一条边就是一个约束，solve() 里你给它装一个力 $\mathbf f$ 和一个 $3\times3$ 的 Hessian $\mathbf H$，然后 $\Delta\mathbf x=\mathbf H^{-1}\mathbf f$ 把端点往满足 rest length 的方向挪一步。Newton 的 rigid AVBD 做的是同一件事的「升维版」：一个 BALL joint 要求两个锚点重合，就像一根 rest length 为 0 的弹簧；一个 REVOLUTE joint 额外锁住两个角向自由度。区别只在变量从 $\mathbb R^3$ 的位置换成了 transform、力/Hessian 要在平移和旋转上一起装配。骨架没变：还是装力、装 Hessian、解一个局部牛顿步、按下降的方向挪。AVBD 在外面再套一层 $\lambda$，把「软弹簧」升级成「能精确满足的硬约束」。

rigid 也要着色：body_color_groups

还记得 particle VBD 那个硬性前提吗——SolverVBD 构造时会检查 model.particle_color_groups 存在，否则按色并行的 Gauss-Seidel sweep 就无从谈起（8.5）。rigid 这一侧是完全对称的：刚体也需要 graph coloring，结果存在 model.body_color_groups 里。只要场景里有刚体由 VBD 积分，这个着色就是必需的。

道理和粒子一模一样：同色的刚体之间不共享 force element（不通过同一个 joint 或 contact 直接耦合），于是可以并行更新；不同色之间串行，拿到 Gauss-Seidel 的收敛。builder.color() 一次把粒子和刚体一起着色，分别填进 particle_color_groups 和 body_color_groups。这是「同一个求解器、同一套并行哲学」最直白的证据——粒子和刚体共用一张着色的世界观。

Newton 当前的一处诚实边界

有一点要照着 Newton 自己的文档说清楚：虽然 AVBD 理论上能强制硬约束，但 Newton 文档目前注明，SolverVBD 对刚体支持的关节是通过软约束（soft constraints）实现的，配合自适应惩罚（adaptive penalties）。也就是说，「让约束冲向真正精确满足」的那条 hard 路径，在当前 Newton 里并没有作为默认承诺被强制起来——能用的是软约束 + 自适应惩罚这条更稳妥的路。这是实现现状与论文理论之间的正常落差，记着即可，别把理论上限当成此刻就能拿到的保证。

几个相关的 solver 参数

不深挖 kernel，但点几个你日后翻 SolverVBD 构造参数时会撞见的名字，好让它们不再陌生。它们都挂在 rigid AVBD 那套 penalty / Lagrangian 机制上：

  • rigid_avbd_gamma：每帧对 penalty 刚度 $k$（以及持久化的对偶量）做衰减的因子 $\gamma$——正是上一节 warm-start 公式里那个把上帧的 $k$ 往回收一点、避免一开始就过硬的衰减系数。
  • rigid_contact_k_start：刚体接触（含 body-particle 接触）penalty 的起始种子 $k_{\text{start}}$，启用 ramping 时从这里起步往上爬。
  • rigid_joint_linear_k_start / rigid_joint_angular_k_start：关节线性与角向约束的 penalty 起始种子，分别管平移方向和旋转方向。joint 的两类自由度刚度量级天然不同，所以拆成两个种子。

这些参数你大概率不需要手动调——上一节说过 AVBD 论文给出的默认值就是「无需调参」的设计目标。它们存在的意义，是当你真要拿 rigid AVBD 去啃一个棘手堆叠或关节链时，知道有这么几个旋钮，且知道每个旋钮拧的是哪一段机制。点到为止，rigid kernel 的内部装配留给好奇心和源码。

进一步阅读

这本 Newton 篇是一份「读懂的地图」，不是终点。如果你想继续往深里走，下面是一份精挑的入口清单（完整、带卷期与 DOI 的参考文献列表在 附录 a-6，这里只给最该先点开的几个）：

  • VBD 原论文（2024）——Vertex Block Descent，Chen / Liu / Yang / Yuksel，ACM TOG 43(4)。这就是 TinyVBD 和 particle 路径的源头，arXiv:2403.06321。全书的 $\min G(\mathbf x)$、block coordinate descent、顶点图着色都从这里来。
  • AVBD 论文（2025）——Augmented Vertex Block Descent，Giles / Diaz / Yuksel，ACM TOG 44(4)，DOI 10.1145/3731195。上一节和这一节 rigid 那套 augmented Lagrangian、penalty ramping、hard/soft 约束都出自它。想搞懂硬约束与高刚度比，读这篇。
  • Newton 文档与代码仓库——求解器契约、Model/State/Control 数据模型、coloring、各 solver 的支持矩阵都在官方文档里；想看 SolverVBD 怎么把 particle VBD 与 rigid AVBD 缝在一个 step() 里，直接读 solver_vbd.py 最实在。
  • Warp 文档——Newton 站在 NVIDIA Warp 之上。看不懂 @wp.kernel / wp.launch / CUDA graph capture 时（7.2），Warp 文档是第一手参考。
  • AVBD 2D 参考实现——Chris Giles 的 savant117/avbd-demo2d，一个干净的二维参考 demo，外加 Utah 的 AVBD 项目页。想像我们读 TinyVBD 那样「逐行读懂一个最小实现」，这是 AVBD 的对应物。

合上这本书

回头看这一整段旅程，它其实只讲了一件事，从一根线讲到一座引擎：

你从 TinyVBD 那根 strand 出发——一根「头发」，~400 行 C++，串行扫一遍顶点，每个顶点解一个 $3\times3$ 的牛顿步。然后我们把它搬上 GPU：Newton 的 particle VBD 把同一套局部牛顿步铺成一块布、用 Warp 的 kernel 按颜色并行扫，加上多种弹性元件、自碰撞、penetration-free。再往前一步，AVBD 用 augmented Lagrangian 补上了 VBD 留下的两块短板——硬约束和高刚度比——并把同一台机器开到了刚体的 joint 与 contact 上。

从 TinyVBD 看 始终是同一个 min G(x)

一路加上去的，全是「机器」（machinery），不是「新物理」。strand → GPU cloth → 硬约束 → rigid，每一步多出来的是并行调度、是图着色、是更多元件类型、是 augmented Lagrangian 这层对偶外壳；但被最小化的东西从头到尾没变——还是那一个增量势能 $G(\mathbf x)=\frac{1}{2h^2}\lVert\mathbf x-\mathbf y\rVert_M^2+E(\mathbf x)$，还是「一步隐式欧拉 = 解一个 $\arg\min G$」。TinyVBD 那根 strand 和 Newton 里翻滚堆叠的刚体，解的是同一道题，只是后者带着更多机械臂去解它。你早在原书 1.2 就握住了这把尺子；这一篇 Newton 之旅，不过是看着同一把尺子把房子盖得更大。

如果你把这两本（原书 + Newton 篇）连起来读完了，那你现在拥有的不是「VBD 怎么实现」这一条知识，而是一种能把任何 position/optimization-based 仿真方法放回谱系的眼光：它在最小化什么？primal 还是 dual？Gauss-Seidel 还是 Jacobi？保留了多少 Hessian？约束是软的惩罚还是硬的 Lagrangian？怎么并行的——着的是顶点图还是约束图？这几个问题，足够你从容地拆解下一个遇到的新方法。
