Block coordinate descent：一次只动一个顶点

前面我们辛苦地把一步隐式欧拉化成了一个最小化问题 $\min_\mathbf{x} G(\mathbf x)$。现在的问题是：怎么解它？Newton 要组装并求解一个大系统，PBD 干脆放弃能量去硬掰约束。VBD 的答案介于两者之间，而且出奇地简单——一次只动一个顶点的 3 个自由度，其余全部按住不动，挨个扫过去。

大想法：把一个大问题拆成一串小问题

$G(\mathbf x)$ 是一个关于 $3N$ 个未知数（$N$ 个顶点，每个 3 个坐标）的函数。Newton 的做法是：把这 $3N$ 个变量看成一个整体，组装一个 $3N\times3N$ 的 Hessian，解一个大线性系统，所有顶点同时往谷底挪一步。这一步很准，但每一步都贵——组装、分解一个大稀疏矩阵，连通性一变还得重新分解。

VBD 反其道而行之。它问：既然目标只是把 $G$ 推下去，我为什么非得一次动所有顶点？我可以固定住除了顶点 $i$ 以外的全部顶点，只把 $G$ 看成「$\mathbf x_i$ 这 3 个自由度」的函数，去优化这一小块，然后换下一个顶点。这就是 block coordinate descent（块坐标下降）：每个「块」是一个顶点的 3 个 DoF，我们一次只在一个块上做优化。

从 GAMES103 看 你解大线性系统时其实也用过这招

课上解隐式积分的大稀疏系统时，你大概率写过 Gauss-Seidel 或 Jacobi 迭代——一次更新一个未知量、扫一遍、再扫一遍。VBD 把这件事抬到了「非线性优化」的层面：它不是在解一个固定的线性方程，而是在非线性能量 $G$ 上做坐标下降。每个顶点的那「一小步」，本身是一个微型的 Newton 步（下面就讲）。所以你可以把 VBD 理解成「在 $G$ 上跑的、每个块带二阶信息的 Gauss-Seidel」。

顶点的局部能量 G_i

把除 $i$ 外的所有顶点钉住，$G$ 里哪些项还会随 $\mathbf x_i$ 变？只有两类：(1) 顶点 $i$ 自己的惯性项；(2) 与 $i$ 相邻的那些弹簧（force elements）的能量——离 $i$ 很远的弹簧此刻是常数，对优化 $\mathbf x_i$ 毫无影响。把这些会变的项收集起来，就是顶点 $i$ 的局部能量：

$$G_i(\mathbf x)=\frac{m_i}{2h^2}\lVert\mathbf x_i-\mathbf y_i\rVert^2+\sum_{j\in\mathcal F_i}E_j(\mathbf x)$$

其中 $\mathcal F_i$ 是与顶点 $i$ 相邻的 force elements（在 strand 里就是 vertAdjacentEdges[i] 那几条边），$\mathbf y_i$ 是顶点 $i$ 的惯性/预测位置，$m_i$ 是它的质量。注意 $G_i$ 只是 $G$ 中「依赖 $\mathbf x_i$」的那一坨——它仍然是真实 $G$ 的一部分，没有任何近似。优化 $G_i$ 关于 $\mathbf x_i$，就等于在「只许动 $i$」的约束下优化 $G$。

坐实一下：G_i 不是近似，是真 G 里依赖 x_i 的那一坨

上面说「$G_i$ 仍是真实 $G$ 的一部分，没有任何近似」——别当口号，用一根最小的 3 顶点 strand 算给你看。顶点 $1,2,3$，边 $e_a=(1,2)$、$e_b=(2,3)$，固定 $\mathbf x_1,\mathbf x_3$，只动 $\mathbf x_2$。完整的 $G$ 是：

$$G=\underbrace{\tfrac{m_1}{2h^2}\lVert\mathbf x_1-\mathbf y_1\rVert^2 + \tfrac{m_2}{2h^2}\lVert\mathbf x_2-\mathbf y_2\rVert^2 + \tfrac{m_3}{2h^2}\lVert\mathbf x_3-\mathbf y_3\rVert^2}_{\text{3 个惯性项}} + \underbrace{E_a(\mathbf x_1,\mathbf x_2) + E_b(\mathbf x_2,\mathbf x_3)}_{\text{2 条边能量}}$$

对 $\mathbf x_2$ 求偏导，逐项看谁活下来：

  • $\mathbf x_1,\mathbf x_3$ 的惯性项：不含 $\mathbf x_2$ → 偏导为 $0$，是常数，丢掉。
  • 不与 2 相邻的边：本例没有（若 strand 更长，比如边 $(3,4)$）→ 不含 $\mathbf x_2$ → 偏导 $0$，丢掉。
  • 顶点 2 的惯性项 $\tfrac{m_2}{2h^2}\lVert\mathbf x_2-\mathbf y_2\rVert^2$、两条相邻边 $E_a,E_b$：含 $\mathbf x_2$ → 留下。

剩下的恰好是 $G_2=\tfrac{m_2}{2h^2}\lVert\mathbf x_2-\mathbf y_2\rVert^2+E_a+E_b$，与上面 $\sum_{j\in\mathcal F_2}$（$\mathcal F_2=\{e_a,e_b\}$）的定义逐字相同。所以 $\nabla_{\mathbf x_2}G_2=\nabla_{\mathbf x_2}G$——优化 $G_2$ 就是在「只许动 2」下优化 $G$，「trust me」变成了「verified」。

每个顶点：一步牛顿

$G_i$ 只有 3 个自由度，这是一个极小的优化问题。VBD 不去精确求它的最小值，而是对它做一步 Newton：算出局部能量的负梯度（也就是作用在 $i$ 上的合力）$\mathbf f_i=-\nabla_{\mathbf x_i}G_i$ 和局部 Hessian $\mathbf H_i=\nabla^2_{\mathbf x_i}G_i$，解一个 $3\times3$ 系统得到位移，然后挪过去：

$$\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i,\qquad \mathbf x_i\mathrel{+}=\Delta\mathbf x_i$$

$3\times3$ 的求解几乎是免费的（甚至能解析地写出来）。$\mathbf f_i$ 和 $\mathbf H_i$ 的具体形式——惯性项加上相邻弹簧的力与 Hessian——我们留到 3.3 一项一项拆开。这里你只要记住节奏：取一个顶点 → 算 $\mathbf f_i,\mathbf H_i$ → 解 $3\times3$ → 更新它的位置 → 换下一个顶点。扫完全部顶点叫一次 sweep；一帧里会做很多次 sweep（TinyVBD 默认 numIterations=100）。

Gauss-Seidel 顺序：用上最新的位置

扫描是顺序的，而且关键在于：当我们处理顶点 $i$ 时，它的邻居里凡是这一遍已经更新过的，我们就用它们的新位置来算 $\mathbf f_i$、$\mathbf H_i$。这正是 Gauss-Seidel 的精神——信息在一遍 sweep 之内就向前传播，收敛比「全用旧位置」的 Jacobi 快。TinyVBD 的 solve() 就是一次这样的 Gauss-Seidel sweep，从 iV=1 扫到末端（顶点 0 被 pin 住、跳过）：

main.cpp · solve()
for (size_t iV = 1; iV < strand.numVerts; iV++)
{
    Mat3 h = Mat3::Zero();
    Vec3 f = Vec3::Zero();
    // 惯性项
    f = strand.vertexMass(iV) * (strand.inertia.col(iV) - strand.vertex(iV)) * dtSqrReciprocal;
    h += strand.vertexMass(iV) * dtSqrReciprocal * Mat3::Identity();
    // 累加相邻弹簧的力与 Hessian
    for (...adjacent edges...) { f += ...; h += h_1_1; }
    Vec3 dx = h.colPivHouseholderQr().solve(f);   // 解 3x3
    strand.vertex(iV) += dx;                        // 立刻就地更新
}

注意最后一行 strand.vertex(iV) += dx 是就地更新 mVertPos，所以下一个顶点 iV+1 在算它自己的力时，读到的就是 iV 的新位置——这就是 Gauss-Seidel。3.4 会讨论它和 Jacobi 的取舍，以及如何用 graph coloring 把它并行化。

慢动作：亲眼看 G 单调下降

这一节的承诺是：每一个这样的局部牛顿步，都让全局 $G$ 单调下降——不是让 $G_i$ 下降而已，是让整个 $G$ 下降。这个「局部 = 全局」的保证，是 VBD 无条件稳定的根，下一节 3.2 会证明它。在那之前，先用下面这个慢动作部件亲眼看一遍：每点一次「更新下一个顶点」，就走一个 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$，左边显示这个顶点当前的 $\mathbf f_i$、$\mathbf H_i$、$\Delta\mathbf x_i$，右边的 $G$ 曲线一路往下走，从不回头。

从 PBD 看 都是 Gauss-Seidel 扫顶点，差别在「扫的时候干什么」

PBD 也按 Gauss-Seidel 顺序一个个处理（不过它处理的是约束，不是顶点），也是就地更新位置、迭代越多越收敛。表面流程几乎一样。差别在于每一步朝哪走：PBD 把粒子投影到约束流形上（把弹簧硬掰回 rest length），它并不盯着任何能量；VBD 则解一个微型牛顿步，明确地把 $G$ 往下推。正因为 VBD 每一步都对着 $G$ 这把尺子，它才有 PBD 给不了的单调下降与稳定性保证。下一节就把这件事说死。

这一节你要带走的

三句话：(1) VBD 用 block coordinate descent 解 $\min G$——一次只优化一个顶点的 3 个 DoF，其余固定，按 Gauss-Seidel 顺序扫；(2) 每个顶点对自己的局部能量 $G_i$ 做一步牛顿 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$，$3\times3$ 求解几乎免费；(3) 关键承诺——每个这样的局部步都让全局 $G$ 单调下降。

下一节我们证明这个承诺：为什么动一个顶点带来的局部能量下降，恰好等于全局能量的下降；以及它如何直接给出 VBD 的无条件稳定。再下一节 3.3 把 $\mathbf f_i$、$\mathbf H_i$ 拆到底。
