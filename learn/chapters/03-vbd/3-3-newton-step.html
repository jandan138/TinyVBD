每个顶点的 3×3 牛顿步：f_i 与 H_i

前两节我们说每个顶点走一步牛顿 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$，并证明了它让全局 $G$ 下降。现在把这个 $\mathbf f_i$ 和 $\mathbf H_i$ 彻底拆开——它们各是两块拼起来的：一块来自惯性，一块来自相邻的弹簧。拆完你会发现，这个「牛顿步」小得几乎可以手算。

两个组件：惯性 + 相邻弹簧

$\mathbf f_i$ 是局部能量 $G_i$ 的负梯度，$\mathbf H_i$ 是它的 Hessian。$G_i$ 由两部分组成——惯性项和相邻弹簧项——所以 $\mathbf f_i$、$\mathbf H_i$ 自然也各分两块：

$$\mathbf f_i = \underbrace{-\frac{m_i}{h^2}(\mathbf x_i-\mathbf y_i)}_{\text{惯性力}} \;-\; \sum_{j\in\mathcal F_i}\frac{\partial E_j}{\partial \mathbf x_i}$$
$$\mathbf H_i = \underbrace{\frac{m_i}{h^2}\mathbf I}_{\text{恒正定}} \;+\; \sum_{j\in\mathcal F_i}\frac{\partial^2 E_j}{\partial\mathbf x_i^2}$$

第一块——惯性项——好理解：它把顶点往预测位置 $\mathbf y_i$ 拉，力的大小正比于偏离量，Hessian 是各向同性的 $\frac{m_i}{h^2}\mathbf I$，一个永远正定的对角块。质量越大、时间步越小，这块越「硬」，把顶点摁在 $\mathbf y_i$ 附近越紧。第二块——相邻弹簧的求和——才是有意思的地方，下面展开。

从相邻边装配：vertAdjacentEdges

$\mathcal F_i$ 这个「与 $i$ 相邻的 force elements」在代码里就是 vertAdjacentEdges[i]——一个边 id 列表，记录哪些弹簧的端点之一是顶点 $i$。在一根普通 strand 里，中间顶点 $i$ 邻接两条边（连向 $i-1$ 和 $i+1$）；若开了 skip spring（近似弯曲），还会多挂一两条连向 $i\pm2$ 的边。装配 $\mathbf f_i$、$\mathbf H_i$ 就是：先放好惯性那一块，再遍历这个列表，把每条相邻弹簧贡献的力和 Hessian 累加上去：

main.cpp · solve() 内层
// 惯性块先放好
f = vertexMass(iV) * (inertia.col(iV) - vertex(iV)) * dtSqrReciprocal;
h += vertexMass(iV) * dtSqrReciprocal * Mat3::Identity();

// 遍历相邻边，逐条累加
for (iNeiE : vertAdjacentEdges[iV]) {
    int edgeId = vertAdjacentEdges[iV][iNeiE];
    IdType v1 = edges[edgeId][0], v2 = edges[edgeId][1];
    Vec3 diff = mVertPos.col(v1) - mVertPos.col(v2);   // d = x_v1 - x_v2
    FloatingType l  = diff.norm();                       // 当前长度
    FloatingType l0 = orgLengths(edgeId);                // rest length
    FloatingType k  = edgesStiffness[edgeId];
    h += k * (Mat3::Identity() - (l0/l) * (Mat3::Identity() - diff*diff.transpose()/(l*l)));
    if (v1 == iV) f += (k * (l0 - l) / l) * diff;        // 力的符号取决于 iV 是哪端
    else          f -= (k * (l0 - l) / l) * diff;
}

每条边只让 $\mathbf f_i$、$\mathbf H_i$ 各加一项——这正是上面求和式 $\sum_{j\in\mathcal F_i}$ 的逐项实现。4.3 会把 solve() 整段逐行再走一遍。

塞进弹簧的力与 Hessian

那「每条边贡献什么」？这正是 1.4 推过的弹簧公式。记弹簧两端为 $a,b$，$\mathbf d=\mathbf x_a-\mathbf x_b$，当前长度 $l=\lVert\mathbf d\rVert$，rest length $l_0$，刚度 $k$。能量、力（对端点 $a$）、Hessian 块分别是：

$$E=\tfrac12 k(l-l_0)^2,\qquad \mathbf f_a = k\,\frac{l_0-l}{l}\,\mathbf d,\qquad \mathbf H_{aa}=k\Big[\mathbf I-\frac{l_0}{l}\Big(\mathbf I-\frac{\mathbf d\mathbf d^\top}{l^2}\Big)\Big]$$

把这两个表达式比对上面代码里的 f += (k*(l0-l)/l)*diff 和 h_1_1 = k*(I - (l0/l)*(I - diff·diffᵀ/(l·l)))，它们一字不差。当顶点 $i$ 处在弹簧的另一端（$b$ 端）时，力反号——代码里 v1 == iV 的判断就是在挑符号。Hessian 块对两端是同一个 $\mathbf H_{aa}$（这个对称弹簧的「自块」对 $a$、对 $b$ 形式相同），所以直接累加即可。

这个弹簧 Hessian 可能不定——而 VBD 不在乎

$\mathbf H_{aa}$ 沿弹簧方向的特征值是 $k$（永远正），垂直方向的特征值是 $k(1-l_0/l)$——当弹簧被压缩（$l变负，弹簧 Hessian 就不定了（1.4 详谈）。Newton/Projective Dynamics 通常要把 Hessian 做 SPD 投影才敢用。VBD 不做这个投影：它指望惯性块 $\frac{m_i}{h^2}\mathbf I$ 这个恒正定的大块把 $\mathbf H_i$ 整体撑回可解的范围，并在万一退化时直接跳过这个顶点。这是 3.5 的主题。

那个 3×3 求解：小到可以解析

装配完，$\mathbf f_i$ 是一个 $3\times1$ 向量，$\mathbf H_i$ 是一个 $3\times3$ 矩阵，求位移就是解

$$\mathbf H_i\,\Delta\mathbf x_i=\mathbf f_i\quad\Longrightarrow\quad \Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$$

这是整套算法里最重的「线性代数」，而它不过是个 $3\times3$。TinyVBD 用 Eigen 的 h.colPivHouseholderQr().solve(f) 来解——稳，但其实一个 $3\times3$ 完全可以用 Cramer 法则解析地写出闭式解，毫无组装大稀疏矩阵、做稀疏分解的负担。这正是 block coordinate descent 的便宜之处：把一个 $3N\times3N$ 的庞然大物，拆成了 $N$ 个互不相干、各自 $3\times3$ 的小步。

（顺带一提：在 strand 这种 1D 结构里，每个顶点只有自己这 3 个 DoF 参与，所以「块」就是 $3\times3$。在体网格里块仍是 $3\times3$——VBD 的块永远是单个顶点的 3 个坐标，无论网格维度。）

把节奏再走一遍

于是顶点 $i$ 的一步，完整地是：

  1. 惯性块：$\mathbf f_i\mathrel{+}=\frac{m_i}{h^2}(\mathbf y_i-\mathbf x_i)$，$\mathbf H_i\mathrel{+}=\frac{m_i}{h^2}\mathbf I$。
  2. 遍历 vertAdjacentEdges[i]：每条边按 1.4 的公式累加弹簧力到 $\mathbf f_i$、弹簧 Hessian 到 $\mathbf H_i$（注意符号取决于 $i$ 是哪端）。
  3. 解 $3\times3$：$\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$。
  4. 就地更新：$\mathbf x_i\mathrel{+}=\Delta\mathbf x_i$，让下一个顶点读到新位置（Gauss-Seidel）。

上一节那个 vbd-sweep 部件里，左侧每步显示的 $\mathbf f_i$、$\mathbf H_i$、$\Delta\mathbf x_i$，就是这四步的实时数值——现在你知道每个数字是怎么算出来的了。

这一节你要带走的

三句话：(1) $\mathbf f_i=\frac{m_i}{h^2}(\mathbf y_i-\mathbf x_i)-\sum_{j}\partial E_j/\partial\mathbf x_i$，$\mathbf H_i=\frac{m_i}{h^2}\mathbf I+\sum_j\partial^2 E_j/\partial\mathbf x_i^2$，惯性块 + 相邻弹簧块；(2) 弹簧块按 1.4 的力/Hessian 公式逐条从 vertAdjacentEdges 装配，符号看 $i$ 在哪端；(3) 最终是一个 $3\times3$ 解 $\Delta\mathbf x_i=\mathbf H_i^{-1}\mathbf f_i$，便宜到可以解析。

下一节我们抬头看并行：这一个个顺序的 Gauss-Seidel 步，怎么用 graph coloring 在 GPU 上并行起来——以及为什么 TinyVBD 自己其实只是老老实实地单线程顺序扫。
